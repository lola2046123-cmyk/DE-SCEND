# 《零号电梯：运行记录》产品设计文档（面向用户）

> **版本**：与当前 Demo（`elevator-rush-demo.html` + `js/game-controller.js`）同步  
> **读者**：玩家、合作方、内容创作者、测试  
> **说明**：前半为体验与承诺；**第 5 章起为与实机一致的规则、数值与操作清单**。若与代码冲突，以仓库内最新逻辑为准。

---

## 1. 产品一句话

**《零号电梯：运行记录》是一款在封闭电梯里进行的、带叙事压迫感的单次博弈体验：**  
你在官僚与超自然交织的「联邦控制局」语境下，用一笔**授权资产**不断**上行**；每次关门升层后随机落入不同**结果带**，资产倍率变化；你可以在决策阶段**风险撤离**锁定收益，或继续赌下一层。偏移指数、同乘**乘客**、环境事件与 **Hiss 入侵倒计时**会不断改写「这一层有多危险」。

---

## 2. 我们要解决的问题（玩家视角）

| 痛点 | 产品的回应 |
|------|------------|
| 纯数字博弈容易麻木 | 电梯相位 UI、工业 HUD、立绘/剪影、Biome 切换，把每次上行做成小型仪式 |
| 读不懂风险时觉得「被耍」 | **偏移指数**条、结果类型（坠毁/负面/正面/双倍）、状态坞提示，让压力可感知 |
| 缺少「人」的张力 | **乘客身份**与**伪装**、**审查**消耗次数、组合羁绊文案 |

---

## 3. 目标用户画像

- 喜欢**短局、多次二选一**（继续 / 撤离）的玩家。  
- 对**新怪谈、工业科幻、《控制》式美学**有共鸣的用户。  
- 能接受**部分信息不透明**（伪装、概率事件）作为玩法的一部分。

---

## 4. 核心体验支柱

1. **工业官僚感**：授权面板、委员会低语、偏移指数、档案式结算面板。  
2. **不稳定的博弈感**：结果带随偏移与乘客漂移；模块次数有限。  
3. **从数字到角色**：乘客剪影、审查揭露、局长/乘客 Subject 立绘（资源允许时）。  
4. **相位化 UI**：引导黑场 → 授权前 → 博弈沉浸态（HUD 双栏保留，部分外围收起）。

---

## 5. 当前 Demo：核心玩法流程（规则）

### 5.1 开局

1. **引导相位（BOOTING）**：开场幕布文案，期间外围控件弱化。  
2. **授权相位（PRE_GAME）**：在**风险资产授权面板**输入额度（**10～99999**）或使用**四档快捷金额**（**100 / 500 / 1K / 5K**），点击 **「初始化运行记录」** 后面板关闭，进入博弈。  
3. **博弈相位（PLAYING）**：显示楼层与资产、偏移指数、井道与轿厢；两侧**金属主钮**（上行 / 风险撤离）与 **Space 指令圆盘**可用。

### 5.2 单次「上行」循环

1. 玩家在 **`IDLE` 或 `DECIDING`** 时触发上行 → 进入升层动画 → 逻辑上进入 **`EVALUATING`**（由控制器在升层结束时结算）。  
2. **到站结算**（一次随机数 `r ∈ [0,1)` 映射到结果带，见 §7）：  
   - 楼层 **+1**；资产先乘 **×1.1**（基线增长），再按结果乘上对应倍率；非坠毁时还可能叠**乘客资本修正**与**波动**（见 §8）。  
   - 累积**偏移指数**（见 §6）。  
   - 处理**乘客下车/上车**（见 §8）。  
   - 每逢 **5 的倍数层**（且 ≥5 层）轮换一条**环境事件**（见 §10）。  
3. 若非 **坠毁**：有小概率触发 **惊喜事件**（见 §9）；并可能标记「本层结束后进入 **Hiss 入侵**」（见 §11）。  
4. **开门展示**结束后调用 `finishReveal`：  
   - **未缓和的坠毁** → 本局 **GAME_OVER**，弹出坠毁档案。  
   - 若本层标记了入侵 → 进入 **`HISS_BREACH`**，**5 秒**倒计时，超时**强制坠毁清零**。  
   - 否则 → **`DECIDING`**，可再次上行或撤离。

### 5.3 撤离与终局

- **风险撤离**：在允许状态下带走**当前资产整数部分**，本局以 **CASHED_OUT** 结束，控制器 **reset**，回到授权界面（连胜计数在 UI 层清零）。  
- **坠毁**：资产归零（或经**应急制动**保留部分，见 §12），展示结果面板后确认，**reset** 回到授权界面。

## 6. 数值设计：偏移指数（Corruption）

| 项目 | 数值/说明 |
|------|-----------|
| 上限 `CORRUPTION.MAX` | **3.0**（UI 条为 `corruption / 3`，显示 0～100%） |
| 对坠毁带的基础放大 | `BOOM_SHIFT = 0.05`：有效 `boomMax` 含一项 `min(corruption × 0.05, 0.20)` |
| 每升一层 | `PER_FLOOR = 0.08`（环境 **CORRUPTION_SURGE** 时该档**翻倍**） |
| 正面结果后 | `ON_POSITIVE = 0.05` |
| 负面结果后 | `ON_NEGATIVE = 0.12` |
| **DOUBLE（超现实双倍）**后 | `ON_DOUBLE = 0.35` |
| 审查等 | VIP 首次审查可 **-0.11**；伪装 **DANGER** 揭露可 **+0.17**（代码内叙事标记） |

**HUD 文案分档**（仅展示用）：`<20%` 稳定；`<42%` 波动；`<65%` 偏高；否则 **临界**。

**贪婪系数**（视觉滤镜等）：`(floor - 1) / 30`，封顶 **1**。

---

## 7. 数值设计：结果带（单次掷骰）

基础阈值（无偏移、无乘客、无环境时的理论下限锚点）：

| 区间 | 基础上限（约） | 结果 | 资产倍率（代码） |
|------|----------------|------|------------------|
| 坠毁带 | `BOOM_MAX ≈ 0.12` + 偏移项 + 乘客/环境修正 | **BOOM** | **×0**（清零，见应急制动例外） |
| 负面带 | 至 `NEGATIVE_MAX` 相关 | **NEGATIVE** | **0.7～0.9** 间线性 |
| 正面带 | 至 `POSITIVE_MAX` | **POSITIVE** | **1.1～1.2** 间线性 |
| 剩余 | — | **DOUBLE** | **×2** |

实际边界由 `computeEffectiveThresholds` 计算：`boomMax`、`negMax` 会随**偏移指数**、**乘客 thresholdMod**、环境 **BAND_DRIFT**（额外 `boom +0.028`）变化；`posMax` 固定为 **0.85**。

## 8. 乘客系统（身份、伪装、审查）

### 8.1 容量与出现

| 项目 | 数值 |
|------|------|
| 同场最多 | **2** 人 |
| 上车概率 | 每层 **40%**（未满员且楼层 ≥ **3**） |
| 下车概率 | 每人每层 **30%** |
| 类型权重 | VIP **35**、SCAMMER **35**、DANGER **30** |

### 8.2 类型概要（机制向）

| 身份 | 阈值倾向 | 资本修正 | 其它 |
|------|----------|----------|------|
| **VIP** | 略降坠毁/负面带 | ×**1.15** | 伪装率 **0** |
| **SCAMMER** | 提高坠毁/负面带 | ×**0.85** | **100%** 伪装成安全形象 |
| **DANGER** | 明显提高坠毁带 | ×**1.0** | **volatile**：结算后再 ×1.25 或 ×0.75（各 60%/40% 一侧）；**50%** 伪装 |

**组合**（多人同时在场时额外修正，节选）：  
- VIP + SCAMMER：阈值再恶化，资本再 ×**1.2**。  
- VIP + DANGER（叙事「对峙」）：阈值结构变化，资本 ×**0.93**。

### 8.3 审查（软盘 · scan 模式）

- 消耗 **1** 次软盘次数；需在 **`DECIDING` 或 `HISS_BREACH`** 且有乘客。  
- 优先揭露仍**伪装**的一位；否则揭露首位。  
- **SCAMMER** 若在伪装状态下被揭穿 → **逃离**（离厢）。  
- **VIP** 首次审查成功 → 偏移 **-0.11**（审计背书）。  
- **DANGER** 且本次为伪装揭露 → 偏移 **+0.17**（希斯相关叙事）。

## 9. 惊喜事件（非坠毁后）

触发条件：`floor ≥ 3`，且本层主结果非 BOOM；另掷随机数。

| 类型 | 基础概率（随贪婪上升） | 效果 |
|------|------------------------|------|
| **硬币雨** | 基础 **5%** × (1+贪婪) | 当前资产约 **+25%** 的整数奖励 |
| **跳层** | 基础 **4%** × (1+贪婪) | 额外 **+1 层**（含该层基线 ×1.1 与每层偏移累积） |

---

## 10. 环境事件（关卡节奏）

每 **5 层**（5、10、15…）在到站时从下列 **3** 条中随机一条设为当前环境（持续到下次被覆盖）：

| ID | 玩家可见信息（状态坞） | 当前控制器内机制 |
|----|-------------------------|-------------------|
| **LIGHTS_OUT** | 灯光熄灭 / 本层倍率封蔽 | **仅展示与叙事**；不改变 `rollEventFromR` 公式 |
| **CORRUPTION_SURGE** | 偏移涨潮 | 结果后累积偏移时 **×2**（与 `PER_FLOOR`、各 `ON_*` 同乘） |
| **BAND_DRIFT** | 阈值漂移 | 坠毁带上限额外 **+0.028** |

---

## 11. Hiss 入侵（HISS_BREACH）

| 项目 | 数值/说明 |
|------|-----------|
| 基础触发 | 非坠毁开门流程结束后，若本层曾标记：楼层 ≥ **5** 且一次随机命中 |
| 基础概率 | **10%**，经乘客身份调制（**DANGER** 升高、**VIP** 降低、VIP+DANGER 同场再升），并夹在约 **2.8%～44%** |
| 倒计时 | **5000 ms**；超时 → **自动坠毁**，资产清零（`autoBoom`） |
| 玩家应对 | **风险撤离**仍可用（与主界面撤离、⌘R 同源逻辑）；**上行**锁定直至脱离该状态 |

界面：**全屏入侵覆层** + 计时；**#core-trigger-override** 为「行政 OVERRIDE」视觉章（与上行键同区，强调入侵态，非单独结算键）。

---

## 12. 物品：局里模块（软盘 floppy-disk）

| 项目 | 说明 |
|------|------|
| 初始次数 | **3** |
| 模式 **scan** | 审查（见 §8.3）；无乘客不可用 |
| 模式 **brake**（应急制动） | 在 **`DECIDING` / `HISS_BREACH`** 且**尚未武装**时可用；武装后直至触发一次坠毁结算前保持 **「应急制动待命」**（状态坞 ⛔） |
| 制动效果 | 下一次 **BOOM** 时资产改为 **本层基线快照后、结果前的 `max(0, floor(快照×30%))`**，并标记 **缓和坠毁**（视觉与叙事不同） |

**入口（当前 Demo）**：  
- **Space** 打开圆盘 → **应急制动**扇区（东侧，高对比样式）。  
- 轿厢**右侧拉杆**（`#manual-brace`）：偏移指数 **>20%** 时显示，点击同调用制动逻辑。

> **引擎预留**：`rerollCurrentFloor`（本层重掷、消耗偏移）存在于 `GameController`，**当前页面未绑定 UI**，玩家暂不可操作。

## 15. 委员会质询（BOSS 弹窗）

- **触发时机**：在 **PLAYING** 下执行**风险撤离**时，若 **连胜 ≥ 2** 且 **30%** 随机命中 → 不立即撤离，先弹出 **委员会质询**。  
- **连胜定义**（UI 层）：连续层结算为 **POSITIVE** 或 **DOUBLE**；坠毁或负面等会清零。  
- **选项**：**继续上行**（关闭弹窗回到决策）；**步出电梯**（执行与撤离相同的结算流程）。

## 18. 交互与可用性原则（对玩家的承诺）

- 关键数字：**楼层 LED**、**资产余额**、偏移条与主钮状态一致。  
- **不可逆消耗**（软盘次数）通过按钮 **disabled** 与圆盘扇区状态表现。  
- **相位锁定**：未授权前不进入升层与撤离主流程。

## 19. 已知局限（Demo 诚实说明）

- 音频依赖 `audio-engine.js`，部分环境可能静音或占位。  
- **本层重掷**未接 UI。  
- 平衡与文案会持续迭代；局域网访问以实际 URL 为准。

## 20. 术语小表

| 术语 | 用户理解 |
|------|----------|
| 上行 | 关门升一层并掷本层结果 |
| 风险撤离 / 步出 | 按当前资产整数结算并结束本局 |
| 偏移指数 | 局内累积，抬高坠毁带并驱动 UI 压迫感 |
| 乘客 | 影响阈值与资本修正的同行者；可有伪装 |
| 审查 | 消耗软盘次数，揭露身份并可能触发逃离/偏移变化 |
| 应急制动 | 武装后下一次坠毁保留约 **30%**（相对该层基线快照） |
| Hiss 入侵 | 限时 **5 秒**，不撤离则可能强制清零 |

## 21. 产品愿景（长期）

每一次开门都像一场**带工业美学的实验**——玩家记住的不只是最后一个数字，还有剪影、扫描、倒计时和「早该收手」的念头。
