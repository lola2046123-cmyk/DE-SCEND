# 《暴富电梯》产品设计文档（面向用户）

> **版本**：与当前 Demo（`index.html` + `js/game-controller.js`，`config/game-events.json._meta.version = 0.05`）同步
> **读者**：玩家、合作方、内容创作者、测试
> **说明**：前半为体验与承诺；**第 5 章起为与实机一致的规则、数值与操作清单**。若与代码冲突，以仓库内最新逻辑为准。

---

## 1. 产品一句话

**《暴富电梯》**（副标：关门再涨价 · 联邦控制局计价规程）是一款封闭轿厢内的单次金额博弈：主标题面向大众动机，局内仍以**联邦控制局**冷面规程呈现。你用一笔**授权资产**不断**上行**；关门升层后弹出 **2~4 只编号货舱**，**亲手选一只开启**——每只货舱背后预绑独立掷骰的结果带，账面随之变化；可在决策阶段**风险撤离**落袋，或继续押下一层。偏移指数、同乘**乘客**、环境事件、**FBC 代决议**与 **Hiss 入侵倒计时**改写「这一层有多贵、有多险」。

---

## 2. 我们要解决的问题（玩家视角）

| 痛点 | 产品的回应 |
|------|------------|
| 纯数字博弈容易麻木 | 电梯相位 UI、工业 HUD、立绘/剪影、Biome 切换，把每次上行做成小型仪式 |
| 读不懂风险时觉得「被耍」 | **偏移指数**条、结果类型（坠毁/负面/正面/双倍）、状态坞提示，让压力可感知 |
| 缺少「人」的张力 | **乘客身份**与**伪装**、**审查**消耗次数、组合羁绊文案、**情报员/清扫员**给出柜号情报 |
| 旧版只能盯着掷骰自动落带 | **封存货舱选盘**：每层亲手开柜；超时则交给委员会代决议，并被偏移与折扣"教训" |

---

## 3. 目标用户画像

- 喜欢**短局、多次二选一**（继续 / 撤离）的玩家。
- 对**新怪谈、工业科幻、《控制》式美学**有共鸣的用户。
- 能接受**部分信息不透明**（伪装、概率事件、情报真伪）作为玩法的一部分。

---

## 4. 核心体验支柱

1. **工业官僚感**：授权面板、委员会低语、偏移指数、档案式结算面板、FBC 代决议横幅。
2. **不稳定的博弈感**：结果带随偏移与乘客漂移；模块次数有限；选盘前的情报真假难辨。
3. **从数字到角色**：乘客剪影、审查揭露、局长/乘客 Subject 立绘（资源允许时）。
4. **相位化 UI**：引导黑场 → 授权前 → 博弈沉浸态（HUD 双栏保留，部分外围收起）。
5. **显式选择优先于自动结算**：玩家在每一层都要"亲手开一只柜"，节奏由你的指尖驱动而不是自动落带。

## 5. 当前 Demo：核心玩法流程（规则）

### 5.1 开局

1. **引导相位（BOOTING）**：开场幕布文案，期间外围控件弱化。
2. **授权相位（PRE_GAME）**：在**风险资产授权面板**输入额度（**10～99999**）或使用**四档快捷金额**，点击 **「启动电梯」**（副行确认计价档案）后关闭面板，进入博弈。
3. **博弈相位（PLAYING）**：显示楼层与资产、偏移指数、井道与轿厢；两侧**金属主钮**（上行 / 风险撤离）与 **Space 指令圆盘**可用。

### 5.2 单次「上行」循环

1. 玩家在 **`IDLE` 或 `DECIDING`** 时触发上行 → 进入升层动画 → 逻辑上进入 **`EVALUATING`**（由控制器在升层结束时结算）。
2. **到站准备**：
   - 楼层 **+1～8（随机跳跃）**；每跳一层各自乘 **×1.1**（基线增长）并累积偏移；跳跃 ≥3 层时 LED 读数触发闪烁特效。
   - 处理**乘客下车/上车**（见 §8）：可能弹出对赌、天使加成或情报员/清扫员的柜号情报。
   - 每逢 **5 的倍数层**（且 ≥5 层）轮换一条**环境事件**（见 §10）。
3. **若不是安全屋楼层 →** 进入 **`LOCKER_SELECTING`** 状态：屏幕中央弹出 **2～4 只封存货舱**（编号 `01..99`），每只柜门已经把对应结果带（POSITIVE / NEGATIVE / DOUBLE / LIQUIDATION）封存进去。你需要在 **倒计时**（与柜数挂钩，区间 6~16 秒）内 **点选一只**，详情见 §5.5。
4. **选盘后**：依次执行
   - **货舱清点**：把本层 outcome 拆成 **3 件子产物**逐件揭露动画（金额按结果类型走 `splitPattern`，文案从 `manifestPool[biome][polarity]` 抽取）；
   - 累积**偏移指数**（见 §6）；
   - 非清算时还可能触发 **惊喜事件**（见 §9）；并可能标记「本层结束后进入 **Hiss 入侵**」（见 §11）。
5. **开门展示**链路末尾由界面调用 `finishReveal` 收口状态：
   - **未缓和的坠毁** → 本局 **GAME_OVER**，弹出坠毁档案。
   - 若本层标记了入侵 → 进入 **`HISS_BREACH`**，**5 秒**倒计时，超时**强制坠毁清零**。
   - 否则 → 直接进入 **`DECIDING`**，可再次上行或撤离。

### 5.3 撤离与终局

- **风险撤离**：在允许状态下带走**当前资产整数部分**，本局以 **CASHED_OUT** 结束，控制器 **reset**，回到授权界面（连胜计数在 UI 层清零）。
- **坠毁**：资产归零（或经**应急制动**保留部分，见 §12），展示结果面板后确认，**reset** 回到授权界面。

### 5.4 生涯成长：累计核销资产、清算与「代决议」档案

你是局长，但 FBC 也在给你**建档**。授权面板顶部的极细统计行（有跨局记录时才会显现）里，**「累计已核销资产」**指你在多局游戏中每一次**成功风险撤离**时被规程**核销并归档**的那笔**撤离到账整数**的**历史总和**——它代表「你曾以合规方式从计价序列中带出多少账面」，而不是单局最高资产。

委员会的**动态评价**（同一行里可能出现的第三句评语）**不直接改写**对局内的概率或阈值，而是档案官僚系统在读你的生涯数字后给出的**叙事反馈**：

| 你的生涯画像（满足其一即可触发对应评语） | 委员会口吻的「动态评价」 |
|-------------------------------------------|-------------------------|
| 累计清算次数 **≥ 3** | 暗示局方对你**稳定性**的担忧。 |
| 累计已核销资产 **≥ ¥5,000** | 暗示你在**财务体量**上已引起总部侧目。 |

若两项同时满足，当前 Demo **优先展示清算相关的担忧句**（更偏风险叙事）。若两项皆未达阈值，则只显示前两行硬数字、不强行塞评语。数字与评价均来自本机 **LocalStorage**，换浏览器或清空站点数据会重置档案。

> **0.05 新档案字段**：`total_committee_overrides`——记录你被 **FBC 代决议** 强制开柜的累计次数（见 §5.6）。该字段已在引擎落地，但 **暂未在授权面板的"动态评价"行直接出现**；它会进入未来的"信用档案"叙事拓展，用以评估你"在选盘窗口里是否果断"。

### 5.5 显式选择：封存货舱选盘（替代 0.04 的"权力制衡"卡牌）

每一层开门后弹出的 **`#locker-selection-overlay`**，是 0.05 的主回路。它做了三件事：

1. **把"看掷骰"改为"开柜门"**：屏幕中央排开 **2~4 只货舱**（养猪期 1–3 楼固定 2 只，行政区 2 只，维护区 3 只，超自然区 4 只），每只都带 `LOT-XX` 编号（`01..99`）。每只柜门 **预绑** 一次独立掷骰的结果带。
2. **倒计时与情报**：浮层底部有进度条倒计时，长度由柜数动态决定（**每柜 4s，截到 6~16s 区间**）。剩余 30% 时进入 **panic 红色脉冲** + 心跳音；若车上有 **情报员 / 清扫员**，可能在浮层副标题区给出 `${locker} 号有油水` 或 `${locker} 号别碰` 一类话术——情报准确率由身份决定（见 §8.4），可能是真也可能是诱饵。
3. **超时即被代决议**：见 §5.6。

操作方式：

| 输入 | 行为 |
|------|------|
| 鼠标点击柜门 | 立即开启该柜（其它柜被禁用、本柜出现 `.opening` 动画） |
| 键盘 **Enter / Space**（聚焦在柜门上） | 同上 |
| 不动手任由倒计时归零 | FBC 代决议（见 §5.6） |

> **开柜后**：本层 outcome 不会再次掷骰，引擎按已封存的结果施加副作用（乘客修正、volatile、对赌乘子等），并进入 §5.7 的"货舱清点"。

### 5.6 FBC 代决议（Committee Override）

倒计时归零仍未选盘时，**委员会代你做决定**：

1. 系统随机高亮一只柜门（红框 `.fbc-override`）+ 低沉盖章音；
2. 按 **62%** 折扣率裁剪本层账面：
   - 若开出来本是净增益 → `账面 = 起点 + 增益 × 0.62`（少赚 38%）；
   - 若开出来本是净亏损 → `账面 = 起点 − 亏损 × 1.38`（多亏 38%）；
3. 偏移指数额外 **+0.45**（与 `corruption.onCommitteeOverride` 同源；这是单步累积里相当重的一档）；
4. 生涯档案 `total_committee_overrides += 1`，并弹出 FBC 横幅 `FBC · 委员会代决议 · 强制开启 LOT-XX`。

**叙事**：你在窗口期没有行使探测员裁量权 → 局方按规程接管 → 你为犹豫付出账面与偏移的双重代价。设计意图是让"不动"成为一种**显式的失败**，而不是默认的安全选项。

### 5.7 货舱清点：把单一金额拆成三段叙事

选定柜门后，屏幕中下部的 **货舱清点条** 会按 `splitPattern` 把本层 outcome 拆成 3 件子产物，按节拍逐件揭露：

| 结果 | 三件分配（默认） | 你的体验 |
|------|------------------|----------|
| POSITIVE | 25% / 35% / 40% | 三件价值递增，最后一件最大；像清点账册时越翻越值 |
| DOUBLE | 20% / 30% / 50% | 节奏更陡，第三件是真正的"裂变红利" |
| NEGATIVE | 30% / 30% / 40% | 三件全损，损失被均摊到三笔条目，逐件让你皱眉 |
| LIQUIDATION | — | 引擎跳过清点（清算节奏不被切碎），直接进入大字金额与清算结果面板 |

每件子产物的**名称与档案号**从 `manifestPool[biome][polarity]` 抽取（行政区抽 `盖章凭证` / `涂改备案表` 等；维护区抽 `黄铜联轴节` / `锈蚀销栓` 等；超自然区抽 `模因护符` / `诅咒账册` 等），加上 `gain / loss / neutral` 三种分极性 SFX，把一个数字拆成"开柜 → 清点 → 总账"的小型仪式。

## 6. 数值设计：偏移指数（Corruption）

| 项目 | 数值/说明 |
|------|-----------|
| 上限 `CORRUPTION.MAX` | **3.0**（UI 条为 `corruption / 3`，显示 0～100%） |
| 对坠毁带的基础放大 | `BOOM_SHIFT = 0.05`：有效 `boomMax` 含一项 `min(corruption × 0.05, 0.20)` |
| 每升一层 | `PER_FLOOR = 0.08`（随机跳跃的每一层各累积一次；环境 **CORRUPTION_SURGE** 时该档**翻倍**） |
| 正面结果后 | `ON_POSITIVE = 0.05` |
| 负面结果后 | `ON_NEGATIVE = 0.12` |
| **DOUBLE（超现实双倍）**后 | `ON_DOUBLE = 0.35` |
| **FBC 代决议后**（v0.05） | `ON_COMMITTEE_OVERRIDE = 0.45` |
| 审查等 | VIP 首次审查可 **-0.11**；伪装 **DANGER** 揭露可 **+0.17** |

**HUD 文案分档**（仅展示用）：`<20%` 稳定；`<42%` 波动；`<65%` 偏高；否则 **临界**。

**贪婪系数**（视觉滤镜等）：`(floor - 1) / 30`，封顶 **1**。

## 7. 数值设计：结果带（每只货舱独立掷骰）

基础阈值（无偏移、无乘客、无环境时的理论下限锚点）：

| 区间 | 基础上限（约） | 结果 | 资产倍率（代码） |
|------|----------------|------|------------------|
| 坠毁带 | `BOOM_MAX ≈ 0.12` + 偏移项 + 乘客/环境修正 | **BOOM** | **×0**（清零，见应急制动例外） |
| 负面带 | 至 `NEGATIVE_MAX` 相关 | **NEGATIVE** | **0.7～0.9** 间线性 |
| 正面带 | 至 `POSITIVE_MAX` | **POSITIVE** | **1.1～1.2** 间线性 |
| 剩余 | — | **DOUBLE** | **×2** |

实际边界由 `computeEffectiveThresholds` 计算：`boomMax`、`negMax` 会随**偏移指数**、**乘客 thresholdMod**、环境 **BAND_DRIFT**（额外 `boom +0.028`）变化；`posMax` 固定为 **0.85**。

> **0.05 起每层会同时生成 N 颗独立 r**（每只柜一颗），最终只有"你选中（或被代决议命中）的那颗"会施加副作用；其余柜的结果会随浮层关闭被丢弃，但仍参与"至少 1 只为正"的兜底约束。

## 8. 乘客系统（5 类身份、伪装、审查、选盘情报）

### 8.1 容量与出现

| 项目 | 数值 |
|------|------|
| 同场最多 | **2** 人 |
| 上车概率 | 每层 **40%**（未满员且楼层 ≥ **3**） |
| 下车概率 | 每人每层 **30%** |
| 类型权重 | VIP **28** / SCAMMER **28** / DANGER **22** / **INFORMER 14** / **CLEANER 8** |

### 8.2 类型概要（机制向）

| 身份（label） | 阈值倾向 | 资本修正 | 伪装 | 其它 |
|---------------|----------|----------|------|------|
| **VIP**（监理员） | 略降坠毁/负面带 | ×**1.15** | 0 | — |
| **SCAMMER**（红圈商人） | 提高坠毁/负面带 | ×**0.85** | **100%** 伪装 | — |
| **DANGER**（希斯潜伏者） | 明显提高坠毁带 | ×**1.0** | **50%** 伪装 | volatile（结果后再 ×1.25 或 ×0.75） |
| **INFORMER**（内线情报员，v0.05） | 中性 | ×**1.0** | 0 | 选盘前 **85%** 概率播报情报，准确率 **78%** |
| **CLEANER**（清扫员，v0.05） | 微降清算/负面 | ×**1.0** | 0 | 选盘前 **45%** 概率播报情报，准确率 **55%**（更易给假情报） |

**组合**（多人同时在场时额外修正，节选）：
- VIP + SCAMMER：阈值再恶化，资本再 ×**1.2**。
- VIP + DANGER（叙事「对峙」）：阈值结构变化，资本 ×**0.93**。

### 8.3 审查（软盘 · scan 模式）

- 消耗 **1** 次软盘次数；需在 **`DECIDING` 或 `HISS_BREACH`** 且有乘客。
- 优先揭露仍**伪装**的一位；否则揭露首位。
- **SCAMMER** 若在伪装状态下被揭穿 → **逃离**（离厢）。
- **VIP** 首次审查成功 → 偏移 **-0.11**（审计背书）。
- **DANGER** 且本次为伪装揭露 → 偏移 **+0.17**（希斯相关叙事）。

### 8.4 选盘情报（INFORMER / CLEANER）

每次进入 LOCKER_SELECTING 之前，引擎会扫描**未伪装**的乘客：

1. 每位掷一次 `lockerHintChance`；命中者按 `lockerHintAccuracy` 取最高的一位作为情报源；
2. 在候选柜中查找首个 POSITIVE/DOUBLE（goodIdx）与首个 NEGATIVE/LIQUIDATION（badIdx）；
3. 50% 偏向播报"好情报"，否则"坏情报"；
4. 准确率掷骰决定情报真假——真情报指向真实柜，假情报指向相反极性柜（无对手柜时随机）；
5. 文案从 `lockerHintCopy.good` / `bad` 池均匀抽取，`${locker}` 替换为两位补零柜号。

> 这意味着 **CLEANER 的情报不一定靠谱**（55% 真）；老练的玩家会把它当作"概率扰动"而不是结论。

## 9. 惊喜事件（非坠毁后）

触发条件：`floor ≥ 3`，且本层主结果非 BOOM；另掷随机数。

| 类型 | 基础概率（随贪婪上升） | 效果 |
|------|------------------------|------|
| **硬币雨** | 基础 **5%** × (1+贪婪) | 当前资产约 **+25%** 的整数奖励 |
| **跳层** | 基础 **4%** × (1+贪婪) | 额外 **+1 层**（在随机跳跃基础上再多跳一层，含基线 ×1.1 与偏移累积） |

---

## 10. 环境事件（关卡节奏）

每 **5 层**（5、10、15…）在到站时从下列 **3** 条中随机一条设为当前环境（持续到下次被覆盖）：

| ID | 玩家可见信息（状态坞） | 当前控制器内机制 |
|----|-------------------------|-------------------|
| **LIGHTS_OUT** | 灯光熄灭 / 本层倍率封蔽 | **仅展示与叙事**；不改变 `rollEventFromR` 公式 |
| **CORRUPTION_SURGE** | 偏移涨潮 | 结果后累积偏移时 **×2** |
| **BAND_DRIFT** | 阈值漂移 | 坠毁带上限额外 **+0.028** |

---

## 11. Hiss 入侵（HISS_BREACH）

| 项目 | 数值/说明 |
|------|-----------|
| 基础触发 | 非坠毁开门流程结束后，若本层曾标记：楼层 ≥ **5** 且一次随机命中 |
| 基础概率 | **10%**，经乘客身份调制（**DANGER** 升高、**VIP** 降低、VIP+DANGER 同场再升），并夹在约 **2.8%～44%** |
| 倒计时 | **5000 ms**；超时 → **自动坠毁**，资产清零（`autoBoom`） |
| 玩家应对 | **风险撤离**仍可用（与主界面撤离、⌘R 同源逻辑）；**上行**锁定直至脱离该状态 |

界面：**全屏入侵覆层** + 计时；**#core-trigger-override** 为「行政 OVERRIDE」视觉章。

## 12. 物品：局里模块（软盘 floppy-disk）

| 项目 | 说明 |
|------|------|
| 初始次数 | **3** |
| 模式 **scan** | 审查（见 §8.3）；无乘客不可用 |
| 模式 **brake**（应急制动） | 在 **`DECIDING` / `HISS_BREACH`** 且**尚未武装**时可用；武装后直至触发一次坠毁结算前保持 **「应急制动待命」**（状态坞 ⛔） |
| 制动效果 | 下一次 **BOOM** 时资产改为 **本层基线快照后、结果前的 `max(0, floor(快照×30%))`**，并标记 **缓和坠毁**（视觉与叙事不同） |

**入口（当前 Demo）**：
- **Space** 打开圆盘 → **应急制动**扇区（东侧，高对比样式）。
- 轿厢**右侧拉杆**（`#manual-brace`）：偏移指数 **>20%** 时显示，点击同调用制动逻辑。

> **0.05 已移除**：旧版的"本层重掷买断（rerollBailout）"与"软盘消耗换重掷"系列。封存货舱选盘的 N 选 1 替代了"翻牌后的二次窗口"。

## 13. 关卡分段：场景 Biome（官卡式阶段）

由**当前楼层**决定背景风格（非独立选关，随进程自动切换），同时**驱动选盘柜数**：

| 楼层 | Biome | 体验描述 | **默认柜数（v0.05）** |
|------|-------|----------|----------------------|
| **1～10** | 行政 / 规整（admin） | 偏冷、秩序感 | **2** |
| **11～25** | 维护 / 闪烁（maintenance） | 灯光不稳、压抑加强 | **3** |
| **26+** | 超自然 / 蠕动（supernatural） | 更强烈的异常感 | **4** |

> 养猪期 1–3 楼会强制把柜数收敛到 **2**，让你在新手期不被"四扇门"压垮。柜数越多，倒计时越长（每柜 4s，区间 6~16s）。

## 15. 委员会质询（BOSS 弹窗）

- **触发时机**：在 **PLAYING** 下执行**风险撤离**时，若 **连胜 ≥ 2** 且 **30%** 随机命中 → 不立即撤离，先弹出 **委员会质询**。
- **连胜定义**（UI 层）：连续层结算为 **POSITIVE** 或 **DOUBLE**；坠毁或负面等会清零。
- **选项**：**继续上行**（关闭弹窗回到决策）；**步出电梯**（执行与撤离相同的结算流程）。

## 18. 交互与可用性原则（对玩家的承诺）

- 关键数字：**楼层 LED**、**资产余额**、偏移条与主钮状态一致。
- **不可逆消耗**（软盘次数、代决议次数）通过按钮 **disabled** 与圆盘扇区状态表现。
- **相位锁定**：未授权前不进入升层与撤离主流程；选盘浮层占用决策权时其它操作禁用。
- **选盘优先于动画**：你的点击立即驱动结算，柜门动画在背景里跑完，**不会因为演出没结束而拖你的节奏**（继承 0.04 期"300ms 快速响应"原则；选盘对应数值为 220ms）。

## 19. 已知局限（Demo 诚实说明）

- 音频依赖 `audio-engine.js`，部分环境可能静音或占位。
- 选盘 SFX 全部为程序化合成（无外部素材依赖），适合早期开发；后续可由音效师替换为采样素材。
- `total_committee_overrides` 已在生涯档案落地，但**尚未在授权面板的"动态评价"行直接体现**。
- 平衡与文案会持续迭代；局域网访问以实际 URL 为准。

## 20. 术语小表

| 术语 | 用户理解 |
|------|----------|
| 上行 | 关门升 1～8 层（随机跳跃）并进入选盘 |
| 风险撤离 / 步出 | 按当前资产整数结算并结束本局 |
| 偏移指数 | 局内累积，抬高坠毁带并驱动 UI 压迫感 |
| 乘客 | 影响阈值与资本修正的同行者；可有伪装；情报员/清扫员还会播报选盘情报 |
| 审查 | 消耗软盘次数，揭露身份并可能触发逃离/偏移变化 |
| 应急制动 | 武装后下一次坠毁保留约 **30%**（相对该层基线快照） |
| Hiss 入侵 | 限时 **5 秒**，不撤离则可能强制清零 |
| 累计已核销资产 | 多局成功撤离到账金额的档案累计，见 §5.4 |
| **封存货舱** | 每层开门后给你的 2~4 只编号柜，每只预绑独立结果带，由你亲手开一只，见 §5.5 |
| **FBC 代决议** | 选盘超时由委员会代你开柜，账面打折 + 偏移加重 + 入档，见 §5.6 |
| **货舱清点** | 选定柜门后将本层 outcome 拆 3 件子产物逐件揭露，见 §5.7 |
| 背景音开关 | 右上角按钮，0.05 起仅开/关两态：关闭只灭背景音乐，事件 SFX 一律保留，见 §16.0 |
