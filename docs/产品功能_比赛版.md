## 一、项目身份 — "壳" 与 "内核"

这不是一个普通的 Roguelike 选择题游戏。它的真正身份是：

> **用《Control》联邦控制局官僚美学包裹的博彩博弈游戏。**

玩家以为自己在"开柜抽奖 / 看故事"，实际上他们正在被一台**完整的 RTP / House Edge / Variance 三件套**精确管理着情绪节拍。

### 1.1 美学壳：联邦控制局 × 新怪谈纵深

参考 *Control · Federal Bureau of Control* 的官僚美学：

- **冷面规程语态**：所有 UI 文本以"§3.1 配额已达成 ×<倍率>"这类档案条目方式呈现，永远不出现"恭喜 / 失败"等情绪词。
- **Oldest House 纵深感**：电梯轿厢、封存货舱、清点货场——三段 3D 微场景以 CSS 透视构建纵深，规整的网格 + 不规则的光晕，制造"不该存在但确实存在"的怪谈感。
- **暖珊瑚 LED + 冷白次级双层色**（v0.06.3）：主读数（FLOOR / `¥` / `#money`）走暖珊瑚 `#f8a59b`，标签 / 配额副数字降级冷白 — mint 仅在配额过线、巨额收益等"极端正反馈"瞬间出现，是**奖励信号而非装饰**。

### 1.2 数学内核：House Edge 博彩对账

下面这张映射表，是这个游戏的真实身份证：

| 博彩术语 | 游戏内对应 | 落地位置 |
|----|----|----|
| **Buy-in 建仓** | 授权金额（¥10 ~ ¥99,999） | 登梯前授权面板 |
| **RTP 玩家回报率** | 综合上行收益率（v0.06 落地后约 117%，长期目标 92–96%） | `_applyBaselineGrowth` × `_applyOutcomeToCredits` |
| **House Edge 庄家边际** | FBC §4.7 流通损耗（4% / 8% / 12%）+ 基线分段衰减 | `fbcEdge.rates` × `baselineGrowth.tiers` |
| **Variance 波动率** | 偏移指数 + 三件子产物分散揭露 | `corruption` × `manifestPool` |
| **Cash-out 撤离** | 风险撤离 / 强制债务撤离 | `cashOut` / `forceDebtCashOut` |
| **Net P&L 净盈亏** | 综合损益结算（核销 − 建仓） | 战绩页第 2 行通栏 |
| **Bonus / Comp** | 配额过线节拍奖励 +1s | `quota.crossedBonusCountdownMs` |
| **Tilt 上头** | 连胜委员会质询 + 倒计时代决议惩罚 | `streakChallenge` / `forceCommitteeOverride` |
| **Compliance 合规** | 失信协议 + 软盘审查 | `forceDebtCashOut` / `auditPassengerLane` |

### 1.3 三条博彩心理学应用

1. **Prospect Theory（损失厌恶 ×2）** — 配额未达即失信协议入档，"输的痛感"被刻意做大，让"撤还是不撤"成为真正的决策。
2. **庄家不送钱** — 每一分授权金额都是真金白银的 buy-in，破产 = 本金归零 + 累计 `total_credits_lost`，账面虚假繁荣无效。
3. **守住 > 赚到** — 基线分段衰减（养肥期 +10% / 节流期 +6% / 冷冻期 +3% / 守钱期 +1.5%）让"上头期"被规程压制；冷冻期每层涨幅几乎被流通损耗吃光，**守住筹码比赚到筹码更难**——这是真实赌场让人上头的核心感受。

## 二、一次对局的节奏

授权 ¥200 → 配额自动核定 ¥1,000（5× 倍率）→ 启动电梯。

楼层 LED 跳了 4 层，账面连乘 ×1.10⁴ → ¥293，偏移涨到 32%。屏幕中央弹出三只编号货舱，倒计时 12 秒。情报员低声："23 号有油水。"——他说真话的概率是 78%，剩下的是直觉。

你点 `LOT-23`。三件子产物逐件揭露：审计票 +¥28、盖章凭证 +¥38、三联印章 +¥52。结果帧底部小字：`// FBC §4.7 已扣 ¥7 · 4% 流通损耗`。账面落到 ¥411，偏移 37%——还有空间。委员会低语在屏幕边缘闪过：「再上一层，离暴富更近——也可能更近破产。」

你按下▲。

**这就是这个游戏一次完整决策的节奏——博彩数学 × 怪谈叙事 × 官僚 UI 在每一秒同时运行，玩家以为自己在"做选择"，实际上每个微决策都被精确计价。**

## 三、技术栈

| 类别 | 选型 | 备注 |
|---|---|---|
| 前端 | **原生 HTML / CSS / Vanilla JS** | 无 React / Vue 框架，零依赖；`index.html` 单文件约 10500 行 |
| 状态管理 | 自研有限状态机（FSM） | `GameController.STATES` × `transitionTo()`，9 个状态、严格单向流转 |
| 数据驱动 | JSON 配置热更新 | `config/game-events.json` 约 700 行；数值改动无需动代码 |
| 持久化 | LocalStorage | `fbc_career_archive_v1` 生涯档案（建仓 / 核销 / 净盈亏 / 清算次数） |
| 音频 | Web Audio API | 程序化合成（齿轮 / 心跳 / 盖章 / 倒计时）；BGM 用 Gemini 生成 |
| 美术 | SVG + CSS Grid + `clip-path` | 货舱 / 卡牌 / HUD 全 CSS 绘制；物品位图由 Midjourney 出 |
| 动画 | CSS `@keyframes` + Web Animations API | 三位一体帧 / 跳层闪光 / 货舱开启 / 货场纵深 |
| 部署 | 静态 HTTP（任意 CDN 即可） | `python3 -m http.server` 本地预览，无后端 |

**代码量统计**：

- `index.html` ~10500 行（CSS 60% / JS 25% / DOM 15%）
- `js/game-controller.js` ~3500 行（状态机 / 概率引擎 / 生涯档案 / 中文本土化口令池）
- `js/audio-engine.js` ~800 行（程序化音效合成）
- `config/game-events.json` ~700 行（数值 / 文案 / 物品池）
- `docs/` 6 份设计文档共 ~3500 行（设计规范 / 全局文案 / 数值汇总 / 版本更新 / 玩家手册 / 比赛版）

**关键工程决策**：拒绝引入任何前端框架。一是为了"单文件开局即玩"的部署简洁；二是为了让 AI 在大段重构时不需要跨多个组件抽象层做心智路由——**AI 在原生 DOM + CSS Grid 上的协作精度远高于在框架抽象层上**。

### 4.2 协作模式 — "一人多角色 + 一个 AI 助手"

我在这个项目里同时扮演四种角色：

1. **博彩游戏数值策划** — 这是我专业领域之外的角色，但博彩思维是这个游戏的核心创意。
2. **资深交互设计师** — 截图、DOM 路径、像素级反馈、跨设备适配判断。
3. **资深前端工程师** — 审 token、提合并条件、跨文件一致性把关。
4. **创意总监 / 编剧** — 定中文本土化基调、否决"不够 FBC"的文案。

而 AI 不是"写手"也不是"工程师"——**它是无限多个不同领域的实习生**：

- 我说"作为资深前端工程师..."→ 它切到工程师人格。
- 我说"你是熟悉博彩行业的策划"→ 它切到博彩顾问。
- 我说"用户角度认为赢钱太容易"→ 它给我**三套方案**让我决断。

### 4.3 一段真实对话（节选）

```
我（数值策划）：玩家角度认为赢钱太容易了，
   感觉钱不值钱。把对弈换成博彩里庄家和玩家的对抗，
   赌场也不可以白送钱，玩家筹码也是真金白银买的。
   怎么样在数值设计上强化这种博弈感？

AI（博彩顾问）：你的问题切中了非常核心的博弈经济学问题。
   最高优先级单一改动：把 baselineGrowth 从固定 1.10
   改为随 stake/quota 比值动态缩减——这一个改动就能解决
   "开局 200 很快到过万"的问题，玩家会感受到
   "守住财富比赚到财富更难"。
   附完整方案 ABC 三选一……

我：执行 改造 ① + ②

AI（前端工程师）：[ 改 CSS / config / JS / 同步 4 份文档 / 跑 lint ]
```

整个项目从启动到 v0.06.3，约 **60 次完整对话循环**，**8 个主要版本迭代**（0.04 → 0.05 → 0.06 → 0.06.1 → 0.06.2 → 0.06.3），单人独立完成。

## 五、AI 是否实质拓展创作边界？

我的回答是：**是。但不是"AI 帮我做了更多"，而是"AI 让我敢做以前不敢做的"。**

### 5.1 拓展的"边界"具体是什么

**第一类：技术天花板的跨越** — 我不是博彩数学专家，但 AI 让我可以**像数值策划一样思考、像专家一样讨论**。整个 House Edge / RTP / Prospect Theory 的迭代，没有 AI 我可能要花 2 周读论文；现在 30 分钟一轮迭代。

**第二类：创作粒度的并联** — 一个人没法同时是"前端 + 数值 + 美术 + 音乐 + 视频"。AI 把这些专业切片**并联到我一个人手里**——我做艺术指导时它是 Midjourney 提示词工程师，我做数值策划时它是博彩顾问，我做前端时它是 senior 工程师。

**第三类：反馈循环的极速闭合** — 截图给 AI → 它能看懂 DOM Path → 改完同步 4 份文档 + 跑 lint。这种"全栈一致性"是单人项目以前几乎做不到的。

### 5.2 拓展的"边界"不包括什么

**AI 替代不了"决断力"。** 当我说"赢钱太容易"时，AI 给了三套方案，**最终选哪个、回退哪个、是不是需要、要不要做、何时做、做到什么粒度**——全是我的判断。

> AI 给方案，我给方向。AI 给方案的实现，我给方案的取舍。

### 5.3 一个具体例子 — v0.06.2 → v0.06.3 LED 色相对调

- **v0.06.2**：AI 听我"绿色太刺眼"反馈，把 LED 全员中性化（默认冷白）。
- **v0.06.3**：我看了一晚发现"主读数失去识别度，太冷"，反过来要求"珊瑚红出现概率更高，白色为次级色"。

**没有 AI**，我也能做这个调整，但要花一晚改 200 行 CSS、4 份文档、跑回归测试。
**有 AI**，30 分钟搞定，且文档自动跟进。

但 **AI 自己想不到"绿色太刺眼"，也想不到"v0.06.2 的中性化错了"**——它只能等我发现、说出来、它执行。

### 5.4 结论

**AI 拓展的是"人的执行半径"，不是"人的判断半径"。**

它让一个人在桌前能完成过去需要 4-5 人才能完成的工作，但**判断力 / 品位 / 决断 / 取舍**——这些 AI 不能给我，只能由我（人类）给。

这个游戏的每一处灵魂都来自我：

- "把博彩内核包在 Control 美学壳里"——这个概念是我的；
- "基线分段衰减 vs FBC 流通损耗 vs 动态配额"该选哪个？是我的；
- "v0.06.2 中性化错了"——这个反思是我的；
- "飞天茅台、安宫牛黄丸放进货舱"——这个本土化是我的；
- "暖珊瑚 + 冷白双层"的色彩组合也是我的。

AI 让我拥有**一个 8 人小工作室的产能**，但灵魂仍是我一个人的。

> **这就是 AI 拓展创作边界的真正含义** — 不是替代人，是让一个有想法的人不再被"我一个人做不完"这件事绊住。
