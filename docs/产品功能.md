## 二、完整流程事件分析

### 2.1 一局完整流程

```
开场幕布 → 授权面板(选额) → [循环开始]
  │
  ├─ 上行 → 随机跳 1-8 层（每层 ×1.1 基线增长）
  │
  ├─ 乘客下车 / 上车（概率触发）
  │
  ├─ 安全屋检查（第 5/10 层首次）→ 若命中 → 净化 or 跳过 → 回到决策
  │
  ├─ 环境事件轮换（每 5 层）
  │
  ├─ 主事件掷骰 → LIQUIDATION / NEGATIVE / POSITIVE / DOUBLE
  │
  ├─ 应急制动检查（若 LIQUIDATION 且已武装 → 缓和）
  │
  ├─ 乘客资本修正 + volatile 波动
  │
  ├─ 偏移指数累积
  │
  ├─ 惊喜事件检查（非 LIQUIDATION 时）：硬币雨 / 跳层
  │
  ├─ Breach 判定（非 LIQUIDATION 时，≥5 层）
  │
  ├─ 开门展示：
  │     ├─ 未缓和的 LIQUIDATION → GAME_OVER
  │     ├─ Breach 标记 → 5 秒倒计时（超时 = 自动清算）
  │     └─ 否则 → DECIDING（继续 / 撤离）
  │
  └─ 撤离时 → 配额检查 → 委员会质询（概率）→ 结算
[循环结束]
```

### 2.2 收益事件（让玩家资产增长的事件）

| # | 收益事件 | 触发条件 | 概率 / 参数 | 收益效果 | 代码位置 |
|---|---------|---------|------------|---------|---------|
| 1 | **基线增长** | 每跳一层自动触发 | 100%（固定） | 资产 ×1.1 | `_applyBaselineGrowth()` |
| 2 | **POSITIVE（正面结果）** | 掷骰 r 落在 [negMax, posMax) | 约 60%（基础，随偏移/乘客波动） | 资产 ×(1.1～1.2) × 楼层杠杆 | `rollEventFromR()` |
| 3 | **DOUBLE（超现实双倍）** | 掷骰 r 落在 [posMax, 1.0) | 约 15%（基础） | 资产 ×2 × 楼层杠杆 | `rollEventFromR()` |
| 4 | **硬币雨（COIN_RAIN）** | 非 LIQUIDATION、≥3 层 | 基础 5% × (1+贪婪) | 资产 +25% | `SurpriseEvent.tryTrigger()` |
| 5 | **跳层（SKIP_FLOOR）** | 非 LIQUIDATION、≥3 层 | 基础 4% × (1+贪婪) | 额外 +1 层 + 基线增长 | `SurpriseEvent.tryTrigger()` |
| 6 | **VIP 乘客加成** | VIP 在场时 | 上车 40% × 权重 35% | 资本修正 ×1.15 + 阈值利好 | `PASSENGER.TYPES.VIP` |
| 7 | **VIP+SCAMMER 组合** | 两者同场 | 条件性 | 资本额外 ×1.2 | `aggregatePassengerModifiers()` |
| 8 | **DANGER 正面波动** | DANGER 在场 volatile 触发 | 60% | 额外 ×1.25 | `processAscendComplete()` |
| 9 | **安全屋净化** | 第 5/10 层首次 | 确定性 | 扣 20% 资产但偏移归零（间接收益） | `resolveSafeNode()` |
| 10 | **VIP 审计背书** | scan VIP 首次 | 条件性 | 偏移 -0.11（降低风险） | `scanPassenger()` |
| 11 | **应急制动缓和** | 已武装 + 触发 LIQUIDATION | 条件性 | 保留约 30% 资产（不归零） | `processAscendComplete()` |
| 12 | **配额穿越** | 资产首次 ≥ 配额 | 单次 | 音效奖励感（无数值效果） | `_quotaCrossed` |

### 2.4 配置文件说明

所有上述事件的概率与数值参数已提取到 `config/game-events.json`，包含 **16 个配置节、80+ 个可调参数**：

| 配置节 | 涵盖参数 |
|--------|----------|
| `thresholds` | 四事件带基础阈值（liquidationMax / negativeMax / positiveMax） |
| `outcomeBands` | 各结果带的资产倍率范围 |
| `corruption` | 偏移指数上限、各场景累积值、放大系数上限 |
| `floorLeverage` | 楼层杠杆系数公式参数 |
| `pigPeriod` | 养猪期保护层数、限幅参数 |
| `baselineGrowth` | 每层基线增长倍率 |
| `floorsJumped` | 每次上行随机跳跃层数范围 |
| `breach` | Hiss 入侵基础概率、乘客调制、时长、概率上下限 |
| `passenger` | 上下车概率、三种类型权重/阈值/伪装、组合加成、volatile 波动 |
| `surprise` | 硬币雨 / 跳层概率、奖励比例、贪婪饱和层 |
| `envEvents` | 触发间隔、三种环境事件各自效果参数 |
| `safeNode` | 安全屋楼层、净化扣费比例 |
| `quota` | 配额目标、压力曲线参数 |
| `item` | 软盘初始数量、制动保留比例 |
| `bossChallenge` | 质询触发连胜阈值、概率 |
| `biome` | 场景换肤楼层区间 |

修改 `config/game-events.json` 即可调整游戏平衡，无需改动代码。
