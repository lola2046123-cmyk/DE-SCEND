## 一、新增系统：局长生涯档案（Director's Dossier）

跨局 **LocalStorage** 持久化，键名 `fbc_career_archive_v1`，由 `GameController` 内 `careerLoad` / `careerMerge` 维护，并在授权面板顶部以极细统计行展示（有数据时淡入）。

| 字段 | 含义 | 写入时机（摘要） |
|------|------|------------------|
| `total_credits_collected` | 累计已核销资产（成功撤离时的 payout 累加） | `cashOut()` |
| `total_floors_climbed` | 累计攀爬层数（与单次上行跳层数同步累加） | `processAscendComplete()` |
| `max_floor_reached` | 历史最高深度（取历史最大楼层） | `processAscendComplete()` |
| `total_liquidations` | 清算次数（破产清算与 Hiss 入侵超时强制清算各记一次） | `finishReveal()`（未缓和清算）、`_autoLiquidate()` |

**代码锚点**：`js/game-controller.js`（`_FBC_CAREER_KEY`、`careerLoad`、`careerMerge` 及上述调用点）；`index.html`（`#fbc-career-stats`、`updateCareerStats()`）。

## 二、UI 进化

### 2.1 「三位一体」结果展示帧（`door-result-frame`）

开门后与主结算链路 **同步淡入**，单行聚合三类信息：

1. **行政名目**：叙事 `tag`（来自 `getOutcomeNarrative`）  
2. **金额变动**：本层 `delta` 的简要符号化展示（增益/损益样式类）  
3. **档案入口**：当本层存在 lore 时显示「查阅异动档案」，与次级流水入口、模态深度阅读形成两级叙事  

**节拍**：展示框 **4 秒** 后自动消隐（`setTimeout(..., 4000)`）；玩家点击 **上行** 或 **撤离** 时由 `hideReceiptSystem()` → `hideDoorResultFrame()` **立即清除**，避免遮挡决策，体现 **操作优先**。

**清算路径**：仍走场景内大字金额闪现与清算弹窗，不占用三位一体框，以免与强反馈冲突。

**代码锚点**：`index.html`（`#door-result-frame`、`showDoorResultFrame` / `hideDoorResultFrame`、开门序列中 `_showAmountAndEffects` 与注释「无需前置叙事预闪，直接同步展示」）。

### 2.2 「干预者协议」选择系统（入场授权）

指 **风险资产授权面板（Deploy / risk-auth-panel）** 内、玩家在 ASCENT 0 阶段对 **规程与杠杆** 的显式确认与选择，语义上对应「干预者」在局前对 FBC 计价规程的介入式签核：

- **规程要点**：`<details>` 可展开区块（注资 / 目标配额 / 增益条款 / 风险与失信条款），与 HUD 许可概要、协议内数字 **联动刷新**（`refreshProtocolLeverageDisplay` 等）。  
- **杠杆选择**：四档预设（LV.01～MAX）+ 手动输入，与 `initialBet` / 局内 stake 对齐。  
- **首登与复登**：`localStorage` 标记首局完成后切换徽章、标题、副文案与「放弃许可」显隐（`applyDeployPanelOnboardingChrome`），形成 **首趟引导式协议面** 与 **标准 FBC 计价规程面** 两套呈现。

**代码锚点**：`index.html`（`#deploy-panel`、`deploy-protocol-details`、预设按钮、`applyDeployPanelOnboardingChrome`）。

## 三、音频架构

### 3.1 音频降维开关（BGM 低通滤波）

- **路由**：`IDLE`、`ASCENDING` 两路 BGM 插槽经 **BiquadFilterNode（lowpass）** → BGM 总线 Gain → Master；其余 SFX 插槽 **直连 Master**，绕过滤波器，保证按键与事件音效清晰。  
- **降维态**：`setBgmDimmed(true)` 将截止频率渐变为 **约 400 Hz**、BGM 总线音量渐变为 **约 30%**；恢复时抬回 **约 20 kHz（等效全频）** 与 **100%**，并用 `cancelScheduledValues` + 线性斜坡降低切换爆音风险。  
- **持久化**：页面提供 **BGM 降维** 切换控件，偏好写入 `fbc_bgm_dimmed_v1`（与 `AudioEngine` 就绪前后状态对齐）。

**代码锚点**：`js/audio-engine.js`（`_bgmFilter`、`BGM_SLOTS`、`setBgmDimmed`）；`index.html`（`#bgm-toggle-btn` 及初始化脚本）。

### 3.2 FBC 氛围音效规范（摘要）

- **插槽制**：具名插槽（`SLOTS`）+ 状态迁移 / 结果 / 惊喜事件到插槽的映射表（`SLOT_MAP_*`），便于按 FBC 事件档案语义对位素材。  
- **分层**：BGM 可走「沉浸降维」总线；清算、配额达成、入侵等 **高优先级反馈** 保持在非滤波 SFX 路径，与 UI 上的 FBC 行政/清算文案层级一致。

## 四、性能与交互响应

### 4.1 300 ms 快速响应（卡牌翻牌）

玩家点选卡牌后，在 **300 ms** 内触发 `hideCardSelection` 与 `handlePostCardReveal()`，与翻牌 CSS 动画 **并行**，注释明确约束：**玩家意志优先，禁止为等待动画而序列化阻塞结算**。

**代码锚点**：`index.html`（卡牌 slot 的 `click` 处理器内 `setTimeout(..., 300)`）。

### 4.2 并行渲染与节拍解耦（开门 reveal）

- **三位一体路径**：非清算时 **跳过** 叙事预闪前置，直接进入金额与特效阶段，减少无意义串行等待。  
- **惊喜事件**：金额与特效主路径后，以配置的 `surpriseAfterDeltaMs`（默认 **300 ms**）**并行** 触发惊喜视觉，不阻塞主 reveal 结算节拍。

**代码锚点**：`index.html`（`_showAmountAndEffects` 内分支与 `surprise` 的 `setTimeout`）。
