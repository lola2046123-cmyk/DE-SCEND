# 《暴富电梯》游戏全局文案梳理

> **版本**：v0.06.3 视觉精修补丁（与 `config/game-events.json._meta.version = 0.06` 对齐；v0.06.2 / v0.06.3 均为纯前端补丁，不涉及 config 字段）。
> **用途**：集中列出玩家可见或辅助无障碍读屏的文案，便于统稿、翻译与合规审查。
> **维护**：改文案时优先对照下表「来源」列；配置项以 `config/game-events.json` 为准，代码内联为兜底或硬编码。
> **品牌**：主标题「暴富电梯」；副标「关门再涨价 · 联邦控制局计价规程」；启动屏 Slogan 见 §3。局内正文保持 FBC 冷面规程（与主标题反差，策略 A）。
>
> **v0.06 本土化基调**：在 FBC 规程腔之上、注入中国情结词汇（恭喜发财 / 落袋为安 / 闷声发大财 / 韭菜 / 鸡蛋分篮 / 加杠杆 等），保持冷面规程语态不破，与「暴富」主品牌形成黑色幽默反差。货舱清点物品全面替换为中国驰名符号（户口 · 学区房 · 飞天茅台 · 安宫牛黄丸 · 田黄石 · 1980 猴票 · 蝴蝶缝纫机 · 永久二八大杠 · BB 机 · P2P 暴雷合同 · 共享单车押金条 等）。
>
> **v0.06.2 增量**：HUD LED 区彻底去 mint（曾全员中性冷白）；移除 `// ALLOC_QUOTA · CLEARED` 装饰标签；菜单"规则"页"金黄发光为正向"改"绿光为正向"。详见 §23。
>
> **v0.06.3 增量**：基于玩家反馈"珊瑚红出现概率应该更高，白色为次级"——LED 主读数（FLOOR / `¥` / `#money` / 任务行 / 沉浸态资产）回到 `--hud-readout` 暖珊瑚（主色复位），冷白降级为标签 / 配额副数字 / mission 标签等"读数附属物"。文案表述无变化，仅视觉色相对调。详见 §24。

## 21. 已知文案分叉（供后续统一时参考）

1. **配额未达成**：`copy.quotaNudge` 与 `showDebtWarning` 默认正文均为局内划扣口径；若配置注入更长模板，宜与许可脚注「信用折偿」一致。
2. **叙事双轨**：引擎 `getOutcomeNarrative`（B）与场景 `formatOutcomeNarrativeHTML`（A）主题相近、句式不同。
3. **对外品牌**：主标题「暴富」仅用于启动/舞台品牌区；局内术语仍以配额、清算、归档为主。
4. **0.04 期"权力制衡 vs 身份卡"分叉已闭合**：UI 与产品文档统一为「封存货舱选盘」（HTML `#locker-selection-overlay` + 文档 §5.5）；旧"`选一张身份卡`"文案已不再出现。
5. **背景音控件**：0.05 起 `aria-label` 以「背景音：开启 / 关闭」为法律与读屏锚点，面向用户文档可继续使用"背景音开关"口径，避免再混入"沉浸模式"。
6. **5 身份标签源**：0.05 起 `IDENTITY_LABELS` 与 `PASSENGER_COMMITTEE` 已在 HTML 内联补齐 5 身份；与配置 `passenger.types.*.label` 形成档案口径 / 简称双轨。后续统稿时建议把审查菜单与立绘标签收敛到 `passenger.types[id].label` 单源（HTML 侧自动叠加 `// ` 前缀），避免再加新身份时双轨漂移。

### 24.4 v0.06.3 三色规则速查（设计师与策划共享）

| 角色 | 用途 | 触发场景 |
|----|----|----|
| **暖珊瑚 `--hud-readout`（主色）** | LED 主读数 / 任务行 / 沉浸态资产 | **常驻**（高频出现） |
| **冷白 `--hud-led-readout-soft`（次级）** | 标签 / 配额副数字 / mission 标签 | **常驻**（陪衬主读数） |
