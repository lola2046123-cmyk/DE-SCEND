## 二、完整流程事件分析

### 2.1 一局完整流程

```
启动屏 → [首次] 开场幕布 → 许可面板（探测杠杆 / 启动或维持现状）
  │
  └─ [循环 · DECIDING]
        │
        ├─ 上行 → 随机跳 1–8 层（每层基线增长）
        │
        ├─ 乘客下车 / 上车（概率触发；可能弹出对赌 confirm / 天使 buff / 情报员或清扫员的柜号情报）
        │
        ├─ 安全屋检查（第 5/10 层首次）→ 净化 or 跳过 → 回到决策
        │
        ├─ 环境事件轮换（每 5 层）
        │
        ├─ generateLockerHand：按 Biome / 养猪期决定柜数 N（2~4），每柜独立掷骰；至少 1 柜为正面带
        │      └─ _setState(LOCKER_SELECTING)：UI 弹出 #locker-selection-overlay + 倒计时
        │            ├─ 玩家点选 → selectLocker(idx) → _resolveLockerSelection(idx, false)
        │            └─ 倒计时归零 → autoSelectLocker() → _resolveLockerSelection(idx, true)
        │                  └─ 账面 × creditsClampRatio · 偏移 += extraCorruption · 生涯档案 +1
        │
        ├─ outcome 落地：偏移累积、惊喜事件（非清算）、Breach 标记、应急制动检查
        │
        ├─ REVEALING：
        │     ├─ runManifestReveal：3 件子产物按 splitPattern 拆金额、按 manifestPool 抽文案、SFX 分极性逐件触发
        │     ├─ 未缓和的 LIQUIDATION → GAME_OVER（清算面板）
        │     ├─ Breach → 倒计时（超时自动清算）
        │     └─ 否则 → DECIDING（继续 / 撤离；软盘 scan/brake）
        │
        └─ 撤离 → 配额/净资产检查 → 质询（概率）→ 结算
```

> SAFE_NODE 节点不进入 LOCKER_SELECTING，直接走原 REVEALING（结算面板内独立处理净化 / 跳过）。

### 2.2 收益类事件（资产上升或风险降低）

| # | 事件 | 触发 / 条件 | 参数位置 | 效果摘要 | 代码锚点 |
|---|------|-------------|----------|----------|----------|
| 1 | 基线增长 | 每升一层 | `baselineGrowth` | 资产 × 倍率 | `_applyBaselineGrowth()` |
| 2 | POSITIVE / DOUBLE | 选盘命中对应带 | `thresholds` · `outcomeBands` | 经 **动态杠杆公式** 增减账面 | `_applyOutcomeToCredits()` |
| 3 | 黄金正面 | 正面带内再掷骰 | `goldenPositive` | 正面倍率升级为超高档（仍 POSITIVE） | `rollEventFromR()` |
| 4 | 硬币雨 / 跳层 | 非清算、`minFloor` 起 | `surprise` | +比例资产 / 额外升层+基线 | `SurpriseEvent` |
| 5 | VIP / 组合加成 | 乘客在场 | `passenger` · `combos` | 资本乘子、阈值修正 | `aggregatePassengerModifiers()` |
| 6 | DANGER 正面 volatile | 60% 分支 | `passenger` | ×`volatilePositiveMult` | `_resolveLockerSelection()` |
| 7 | 天使投资人 | 未伪装 VIP 上车 | `angelInvestor` | 存续层数内抬高正面 rate | `_angelLift` |
| 8 | 对赌协议（收益侧） | 签署后存续层内 | `speculatorProtocol` | 正面/翻倍 **delta × gainMult** | `_speculatorContract` |
| 9 | 连击盲信 | 短窗口内再次上行 | `comboBlindConfidence` | 本层 delta ×`gainMult`（隐性 quota 压力） | `_blindConfidenceActive` |
| 10 | 安全屋净化 | 指定层首次 | `safeNode` | 付比例换偏移清零 | `resolveSafeNode()` |
| 11 | VIP 审查背书 | scan VIP | `corruption.onVipAuditRelief` | 偏移回退 | `scanPassenger()` |
| 12 | 应急制动缓和 | 武装 + 清算 | `item.floppyDisk.brakeRetainRatio` | 不按零清算 | `mitigated` |
| 13 | 选盘正面兜底 | `generateLockerHand` 最后一柜替换 | `lockerSelection.minBidGapPositive` | 至少 1 只货舱命中 POSITIVE/DOUBLE | `generateLockerHand()` |
| 14 | 情报员真情报 | 选盘前 | `lockerHintAccuracy` | 高准确度指向真实正/负柜 | `_computeLockerHint()` |
| 15 | 配额穿越 | 首次资产 ≥ 配额 | — | 音效反馈 | `_quotaCrossed` |

### 2.4 配置文件说明（v0.05）

`config/game-events.json` 顶层除 `_meta` 外，主要配置节包括（修改后无需改代码即可调平衡）：

| 配置节 | 涵盖内容 |
|--------|----------|
| `thresholds` · `outcomeBands` | 四事件带阈值与各带倍率 |
| `corruption` | 偏移上限、各场景累积、清算带放大上限、**`onCommitteeOverride`** |
| `floorLeverage` · `pigPeriod` · `baselineGrowth` · `floorsJumped` | 杠杆、养猪期、基线、跳层 |
| `breach` | Hiss 入侵 |
| `passenger` | 5 身份权重、修正、伪装、组合、volatile、**`lockerHintChance` / `lockerHintAccuracy`** |
| `surprise` · `envEvents` · `safeNode` · `quota` | 惊喜、环境、安全屋、配额压力 |
| `speculatorProtocol` · `angelInvestor` | 对赌协议、天使增益 |
| `goldenPositive` · `comboBlindConfidence` | 肥尾、连击 |
| `suspense` | 清算/负面揭晓阻塞毫秒 |
| `copy` | 配额拦截、对赌等文案模板 |
| `item` | 软盘次数与制动留存比例 |
| `bossChallenge` | 质询触发 |
| `uiPacing` | 叙事/金额/选盘/清点等 UI 节拍（含 `lockerOverlayShowMs` / `manifestItem*Ms`） |
| `biome` | 场景楼层区间 |
| **`lockerSelection`** | 柜数 / 倒计时 / 编号区间 / 至少一柜为正的兜底约束（v0.05 新增） |
| **`manifestReveal`** | 货舱清点：3 件子产物拆金额模式（v0.05 新增） |
| **`manifestPool`** | 子产物文案池（biome × 极性，v0.05 新增） |
| **`fbcOverride`** | 委员会代决议折扣 / 偏移追加 / 横幅与回音文案（v0.05 新增） |
| **`lockerHintCopy`** | 情报员/清扫员的好坏柜话术（v0.05 新增） |

> **0.05 已删除**：`cardSelection` / `rerollBailout` / `chainLightning` / `assetFreeze`。即使旧配置或旧 LocalStorage 残留这些键，引擎会静默忽略。
