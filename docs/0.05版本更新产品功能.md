# 《暴富电梯》0.05 版本产品功能表 & 事件分析（里程碑更新）

> **版本**：0.05
> **生成日期**：2026-04-26
> **说明**：本文档在 `docs/0.04版本更新产品功能.md` 之上记录 **0.05 里程碑** 的破立式改动；概率与数值仍以 `config/game-events.json`（`_meta.version` 同步为 `0.05`）为准。

---

## 一、立项目标：把「单点掷骰」改造为「显式选盘」

0.04 之前，开门后玩家几乎只能看着一颗随机数自动落入某个结果带；卡牌选取虽给出 3 张身份卡，但**仍是发牌后再翻**，玩家的"我"在节拍上长期处于被动。0.05 把这条主回路彻底重写为 **「封存货舱选盘」**：

- 玩家在每一层开门后从 **2~4 只编号货舱** 中亲手开 **一只**；
- 每只货舱背后预绑定**独立掷骰**的结果（POSITIVE / NEGATIVE / DOUBLE / LIQUIDATION）；
- 倒计时归零未选 → **FBC 代决议** 自动选盘并按 ratio 折扣账面、追加偏移、累计生涯档案；
- 选定后再进入 **货舱清点** 三件子产物逐项揭露动画，把单一金额拆成三段叙事。

> 这一改动同时**废除**了 0.03–0.04 期间的 **卡牌选取（cardSelection）** / **重掷买断（rerollBailout）** / **连环爆雷（chainLightning）** / **资产冻结（assetFreeze）** 四套子系统：玩家主动性已经被前移到选盘步骤，下游不再需要这些"翻牌补偿"机制。

## 二、新系统：封存货舱（Locker Selection）

### 2.1 数据流与状态机

```
EVALUATING → processAscendComplete()
   │
   ├─ SAFE_NODE 楼层 → 直接 REVEALING（沿 0.04 路径）
   │
   └─ 普通楼层 → generateLockerHand(count, quotaShift)
                    └─ N 个候选 outcome（独立 r、独立柜号）
                    └─ _setState(LOCKER_SELECTING)
                    └─ 派发 onLockerHand / onLockerHint
```

新增控制器状态 **`LOCKER_SELECTING`**（`STATES.LOCKER_SELECTING`）；`startAscend` 会拒绝在该状态下重入。

### 2.2 引擎接口

| 方法 / 静态导出 | 用途 |
|---|---|
| `GameController.STATES.LOCKER_SELECTING` | 选盘期状态值 |
| `GameController.LOCKER_CFG` | `lockerSelection` 配置缓存 |
| `GameController.FBC_OVERRIDE_CFG` | `fbcOverride` 配置缓存 |
| `GameController.LOCKER_HINT_COPY` | `lockerHintCopy` 缓存 |
| `GameController.generateLockerHand(ctrl, count, quotaShift)` | 静态：生成 N 个候选（含柜号去重 + 兜底顺序补齐，避免确定性 RNG 死循环） |
| `GameController.biomeForFloor(floor)` | 静态：按 `biome.ranges` 推导 `admin / maintenance / supernatural` |
| `prototype.selectLocker(idx)` | 玩家主动选盘 |
| `prototype.autoSelectLocker()` | 倒计时归零后由 UI 调用，标记 `committeeOverride = true` |
| `prototype._computeLockerHint(candidates)` | 情报员/清扫员上车时生成柜号情报（准确率随身份配置） |

### 2.3 配置（`config/game-events.json` · 新增节）

| 节 | 主要字段 | 作用 |
|---|---|---|
| `lockerSelection` | `lockerCountByBiome`（admin=2 / maintenance=3 / supernatural=4） · `countdownPerLockerMs=4000` · `countdownMinMs=6000` · `countdownMaxMs=16000` · `lockerNumberMin=1` · `lockerNumberMax=99` · `panicTickThresholdMs=3000` · `minBidGapPositive=1` | 柜数 / 倒计时 / 编号区间 / 至少一柜为正面的兜底约束 |
| `manifestReveal` | `splitPattern.{POSITIVE/DOUBLE/NEGATIVE/LIQUIDATION}` · `fallbackSplit` · `itemsPerReveal=3` | 货舱清点拆三件的金额分配模式 |
| `manifestPool.{biome}.{positive/negative/neutral}` | `id / name / tag / weight` | 子产物文案池（admin/maintenance/supernatural × 三极性） |
| `fbcOverride` | `creditsClampRatio=0.62` · `extraCorruption=0.45` · `banner` · `messages[]` | 代决议折扣比、追加偏移、横幅与回音文案 |
| `lockerHintCopy` | `good[]` / `bad[]` | 情报员/清扫员的好坏柜口播话术 |

> 养猪期（`PIG_PERIOD.maxFloor=3`）会把柜数强制收敛到 **2**，倒计时同步缩短到 6s 下限，避免新手开局被"满桌选项"压垮。

### 2.4 候选生成保证

`generateLockerHand` 内置三层兜底：

1. **掷骰循环**：最多 `count*6` 次重抽，避免极端 RNG 时反复落入清算带；
2. **正面兜底**（`minBidGapPositive=1`）：若 N 次抽完仍无 POSITIVE/DOUBLE，强制把其中一柜替换为正面带的拟造 outcome；
3. **柜号兜底**：随机抽取 `numberMin..numberMax` 范围内不重复的柜号，若 RNG 退化（确定性测试场景），fallback 到顺序补齐，**保证函数总能返回所需柜数**。

## 三、UI / 节拍：选盘浮层与货舱清点

### 3.1 选盘浮层（`#locker-selection-overlay`）

DOM 结构：

```
.locker-overlay__classification   FBC · 封存货舱 · 待开启
.locker-overlay__title            选一只货舱开启
.locker-overlay__subtitle         超时未选 · 委员会代决议（账面按 62% 折扣留存）
.locker-overlay__hint             [情报员/清扫员动态情报，含柜号]
.locker-grid                      LOT-XX 卡片 × N
.locker-countdown                 进度条 + 文案 "FBC · 倒计时"
```

- **点击 / Enter / Space**：调用 `handleLockerClick(idx, slot)`；选中柜门加 `.opening` 动画，其余加 `.disabled`，220 ms 后驱动 `game.selectLocker(idx)`。
- **倒计时**：`startLockerCountdown(totalMs)`；`ratio ≤ 0.3` 触发 `panic` 类（红色脉冲）+ **一次性** `playLockerPanic()` SFX；归零时随机柜门加 `.fbc-override` 类、240 ms 后调用 `game.autoSelectLocker()`。

### 3.2 货舱清点（`#manifest-strip`）

`runManifestReveal(outcome, deltaMoney, biome)`：

1. 根据 `outcome.kind` 取 `splitPattern`（默认 `[0.33, 0.33, 0.34]`）；
2. 为每件子产物计算 `delta = round(deltaMoney * portion)` 与极性（gain / loss / neutral，最后一件吸收四舍五入误差）；
3. 从 `manifestPool[biome][polarity]` 内权重无放回抽 3 件文案；
4. 按 `uiPacing.manifestItemStaggerMs / manifestItemHoldMs / manifestRevealStartMs` 的节拍依次为每行加 `revealed / gain / loss` 类，**每件触发 `playManifestRevealItem(polarity)` 事件音**；
5. 返回总时长 `startDelay + N*stagger + hold`，调用方据此延后 `finishReveal`。

清算 / 安全屋分支跳过清点（避免冲淡死亡或安全节奏）。

## 四、乘客身份扩展：5 类 + 选盘情报

`passenger.types` 在原 VIP / SCAMMER / DANGER 之上新增两类，权重重新平衡：

| 身份 | 权重 | label | 阈值 / 资本修正 | 选盘相关 |
|---|---|---|---|---|
| **VIP** | 28 | 监理员 | `liquidationShift=-0.04` · `creditsMod=1.15` | — |
| **SCAMMER** | 28 | 红圈商人 | `liquidationShift=+0.06` · `creditsMod=0.85` · 100% 伪装 | — |
| **DANGER** | 22 | 希斯潜伏者 | `liquidationShift=+0.08` · volatile | — |
| **INFORMER**（新） | 14 | 内线情报员 | 阈值 / 资本中性 | `lockerHintChance=0.85` · `lockerHintAccuracy=0.78` |
| **CLEANER**（新） | 8 | 清扫员 | `liquidationShift=-0.01` · 阈值/资本近中性 | `lockerHintChance=0.45` · `lockerHintAccuracy=0.55` |

`_computeLockerHint` 在选盘前掷骰：命中后从 `lockerHintCopy.good` / `bad` 抽一条话术，并按 `lockerHintAccuracy` 决定情报指向真实柜或反指；UI 在浮层 `.locker-overlay__hint` 直接显示。

`index.html` 的 `PASSENGER_COMMITTEE` 与 `IDENTITY_LABELS` 已同步补齐 5 身份的上车口令与审查档案标签：

- **INFORMER** 上车 → 委员会播报 `线人入场：货舱档案已抄送。`；档案标签 `// 内线情报员`。
- **CLEANER** 上车 → 委员会播报 `清扫员到位：现场痕迹半数被抹。`；档案标签 `// 清扫员`。

两条口令保持 FBC 冷面规程语态：INFORMER 暗示"档案已抄送"对应高准确率情报；CLEANER 暗示"痕迹半数被抹"对应配置中较低的 `lockerHintAccuracy = 0.55`，与机制语义一致。

## 五、FBC 代决议（Committee Override）

### 5.1 触发与处置

倒计时归零时 UI 调用 `autoSelectLocker()`，引擎随机选一柜并标记 `outcome.committeeOverride = true`：

1. **账面折扣**：盈亏经乘客修正后再裁剪——
   - 净增益 → `credits = floorBefore + gain * ratio`（`creditsClampRatio=0.62`）
   - 净亏损 → `credits = floorBefore - loss * (1 + (1 - ratio))`（亏损反向放大）
2. **偏移指数**：`+= extraCorruption`（默认 +0.45，比正面/负面单步累积都重，且与 `corruption.onCommitteeOverride` 同源）；
3. **生涯档案**：`careerMerge({ committeeOverride: true })` → `total_committee_overrides += 1`；
4. **横幅** `fbcOverride.banner` 经 UI 委员会闪条 `flashCommitteePassengerDirective` 推送；正文池 `fbcOverride.messages` 留作未来 NPC 回音文本扩展。

### 5.2 生涯档案字段更新

`_FBC_CAREER_KEY = 'fbc_career_archive_v1'` 仍兼容旧存档，新增字段：

| 字段 | 含义 | 写入时机 |
|---|---|---|
| `total_committee_overrides` | 累计被 FBC 代决议次数 | `_resolveLockerSelection` 内 `careerMerge` |

`careerLoad` / `careerMerge` 同步处理新字段；缺键回退 0，跨版本读旧档不会爆。

## 六、音频架构

### 6.1 BGM 切换：三态 → **两态**

0.04 的 `normal / dimmed / muted` 三态在 0.05 简化为 **`on / off`**：

- **on**：BGM 总线 `gain=1.0`、`themeBgmEl.volume=0.38`、master 全开；
- **off**：BGM 总线 `gain≈0`、`themeBgmEl.pause()`、**master 仍维持 `MASTER_LEVEL`**——所有事件 / 操作 SFX 一律保留。

新增 `AudioEngine.setBgmMuted(boolean)` / `isBgmMuted()`：仅作用于 `_bgmBus.gain`（与 `setBgmDimmed` 解耦），并通过 `_ensureContext` 应用预 Context 时记录的静音意图。`applyEngineMode(mode)` 进入 off 时还会兜底清掉历史残留的 `setMasterMuted(true)` 与 `setBgmDimmed(true)` 状态。

旧存档迁移：`'normal'/'dimmed'` → `'on'`，`'muted'` → `'off'`；同时清除 `fbc_bgm_dimmed_v1` 旧标志。

### 6.2 选盘 SFX（新增 5 类）

全部在 `js/audio-engine.js` 内以 Web Audio 程序化合成（不依赖外部素材，**直连 master 总线，绕过 BGM 滤波**），保证 BGM 关闭时仍清晰可闻：

| 时机 | 方法 | 设计要点 |
|---|---|---|
| 选盘浮层弹出 | `playLockerSelectionEnter()` | 金属插销"咔" + 146→196 Hz 缓推悬念音 |
| 玩家点选某柜 | `playLockerOpen()` | 720→380 Hz 解锁 click + 带通扫频白噪滑动金属 + 660 Hz 三角波共鸣 ping |
| 倒计时进入 panic | `playLockerPanic()` | 双拍心跳重音（120→58 Hz sine），首次跨阈值时一次性触发 |
| FBC 代决议盖章 | `playFbcOverrideStamp()` | 110→48 Hz 低 square 重落 + 公文章纸面摩擦 + 阴森锯齿余音 |
| 货舱清点逐件揭露 | `playManifestRevealItem(polarity)` | gain：880→1320 Hz 清亮上扬；loss：220→96 Hz 短促下沉；neutral：640 Hz 轻 tick |

UI 接入点：`showLockerOverlay` / `handleLockerClick` / `startLockerCountdown` / `runManifestReveal`。

## 七、被废除的 0.04 子系统

| 系统 | 0.05 处置 | 替代品 |
|---|---|---|
| `cardSelection` | 完全移除（DOM / JS / 配置 / `CARD_SELECTION` 状态） | `lockerSelection` |
| `rerollBailout` / `rerollCurrentFloor` | 删除（引擎与配置） | 选盘本身即"决策前的二次窗口" |
| `chainLightning`（连环爆雷 + 跨局负债） | 删除（含 `localStorage` 链债读取） | — |
| `assetFreeze`（资产冻结） | 删除（含 `_frozenPrincipal` / `_frozenUnlockFloor`） | — |

`config/game-events.json._meta.description` 已显式注明上述删减；旧存档读取相关键时静默忽略，不再触发引擎分支。
