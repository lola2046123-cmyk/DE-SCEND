# 《暴富电梯》游戏全局文案梳理

> **版本**：v0.07.3 心流修复 + UX 打磨（基于 v0.07.2 庄家人格化；本次无经济数值变更，仅 UI/UX/音效层更新）。
> **用途**：集中列出玩家可见或辅助无障碍读屏的文案，便于统稿、翻译与合规审查。
> **维护**：改文案时优先对照下表「来源」列；配置项以 `config/game-events.json` 为准，代码内联为兜底或硬编码。
> **品牌**：主标题「暴富电梯」；副标「关门再涨价 · 联邦控制局计价规程」；启动屏 Slogan 见 §3。局内正文保持 FBC 冷面规程（与主标题反差，策略 A）。
>
> **v0.06 本土化基调**：在 FBC 规程腔之上、注入中国情结词汇（恭喜发财 / 落袋为安 / 闷声发大财 / 韭菜 / 鸡蛋分篮 / 加杠杆 等），保持冷面规程语态不破，与「暴富」主品牌形成黑色幽默反差。货舱清点物品全面替换为中国驰名符号（户口 · 学区房 · 飞天茅台 · 安宫牛黄丸 · 田黄石 · 1980 猴票 · 蝴蝶缝纫机 · 永久二八大杠 · BB 机 · P2P 暴雷合同 · 共享单车押金条 等）。
>
> **v0.06.2 增量**：HUD LED 区彻底去 mint（曾全员中性冷白）；移除 `// ALLOC_QUOTA · CLEARED` 装饰标签；菜单"规则"页"金黄发光为正向"改"绿光为正向"。详见 §23。
>
> **v0.06.3 增量**：基于玩家反馈"珊瑚红出现概率应该更高，白色为次级"——LED 主读数（FLOOR / `¥` / `#money` / 任务行 / 沉浸态资产）回到 `--hud-readout` 暖珊瑚（主色复位），冷白降级为标签 / 配额副数字 / mission 标签等"读数附属物"。文案表述无变化，仅视觉色相对调。详见 §24。
>
> **v0.06.4 增量**：柜号情报 `lockerHintCopy` 按 `passenger.identity` 拆为 INFORMER（FBC 内部情报腔）/ CLEANER（市井 / 工人 / 老韭菜腔）专属子池，修复"情报员低声 + 包工头叼着烟"挂在同一乘客的人格漂移。同时补录三处文案档漏录（§7.5 FBC 流通损耗注脚 · §6 INFORMER/CLEANER 状态坞现状 · §20.1 战绩档案页 5 行布局）。详见 §25。
>
> **v0.07 增量**：信用评级 v1 跨局张力系统落地——连续清算驱动 B/C/D 降档，连续撤离驱动 AAA 升档；授权面板新增 `#deploy-credit-rating-line` 信用评级行；buy-in 输入上限按 `initialBetMaxMult` 动态钳位。详见 §26。
>
> **v0.07.1 增量**：浮盈折现 + 撤离税（House Edge ⑤⑥）——正向收益叠加 surplusDiscount 折现层、撤离超额部分加收 evacuationTax；三位一体结果框新增 §6.2 / §8.1 注脚行。详见 §27。
>
> **v0.07.2 增量**：庄家人格化 · 委员会三触发——大赢反应低语、连胜探测协议（edgeBoost +2%）、高偏移加码（countdown ×0.9）；授权面板新增探测协议行 `#deploy-pressure-protocol-line`。详见 §28。
>
> **v0.07.3 增量**：心流修复 + UX 打磨——① 战绩空状态缺省图（实心块电梯井像素图示）；② 数值用色全局翻转（红涨绿跌 · 中国股市惯例）；③ 货舱清点心流修复（总等待 8.16s→3.28s）；④ 货舱清点点击跳过（holding 阶段任意点击直接切下一张）；⑤ 音效新增 `playScanPrint`（三位一体展示框淡入扫描音）+ Stamp `playbackRate` 微随机化（0.95×–1.05×）。详见 §29。

### 8.1 情报员 / 清扫员的柜号情报（`config.lockerHintCopy` · v0.06.4 按身份拆池）

> **v0.06.4 改造**：原 12 条混合池（INFORMER 与 CLEANER 共用 good/bad 各 6 条）已按 `passenger.identity` 拆为身份专属子池，纠正"FBC 情报员低声"与"包工头叼着烟"挂在同一乘客口中的人格漂移问题。结构详见 `config/game-events.json · lockerHintCopy.byIdentity`，引擎路由见 `js/game-controller.js · _pickLockerHintPool()`。
>
> **兜底**：顶层 `lockerHintCopy.good / .bad`（仍保留 12 条）作为 fallback——子池缺失或未识别身份时回落，零回归风险。**当前 INFORMER / CLEANER 必命中子池**，不会触发兜底；兜底主要服务于旧档案 / 未来扩展身份。

#### 8.1.1 INFORMER（内线情报员） · FBC 内部情报腔

> **腔调**：侦测 / 编号 / 规程语态，官僚冷感；信息精准（`lockerHintAccuracy = 0.78` 真）但生硬。

## 27. v0.07.1 浮盈折现 + 撤离税（House Edge 升级 ⑤⑥）

> **核心命题**：完成数值经济路线图的最后两阶段，把高资产玩家的 RTP 从 117% 推回 98–103% 设计目标区间。两个机制同方向但语义不同：**浮盈折现是"奢侈税"**（仅 stakeRatio ≥ 3 的高资产玩家在每层正向 delta 上累进抽水），**撤离税是"分红税"**（仅 cashOut 路径对超出配额的部分征收一次性手续费）。
> **配置层**：`config/game-events.json · surplusDiscount` / `evacuationTax`；**引擎挂钩**：`_applyOutcomeToCredits`（折现）/ `cashOut`（撤离税）；**UI 渲染**：`#drf-discount`（折现注脚）/ 结果屏成功撤离 stat-row（手续费行）。

### 27.1 浮盈折现（House Edge 升级 ⑤）

| 项 | 规则 |
|---|---|
| 触发条件 | `delta > 0` 且 `stakeRatio = stake / quota >= 3` |
| 分档 | stakeRatio ∈ [3, 6) → ×0.88（节流期）<br>stakeRatio ≥ 6 → ×0.72（守钱比赚钱难） |
| 应用顺序 | speculator/blindConfidence 放大乘子 → **浮盈折现** → fbcEdge 流通损耗（串行） |
| 数据写入 | `outcome._surplusDiscountMult` / `outcome._surplusDiscountAmount` |
| 兼容回退 | `surplusDiscount.enabled = false` 整段 no-op，乘子恒为 1 |

**结果框注脚**（`.door-result-frame__discount` · 仅 discountAmount > 0 时显示）：

```
// FBC §6.2 已折 ¥{amount} · {pctOff}% 浮动折现
```

> 与 `.door-result-frame__edge`（FBC §4.7）同字号、同色相、同动效；位置在流通损耗注脚**上方**（折现先于流通损耗执行，呈现顺序也对应）。两行同时存在时第二行间距收紧 0.18rem 防视觉拥挤。

> **设计意图**：让玩家在 stakeRatio 跨过 3.0 / 6.0 时**清晰感知到"账面增速被打折"**——前两层一切正常，第三层突然 +¥320 变 +¥282，玩家自然会想"我是不是太富了"——这正是审计提到的"守钱比赚钱难——落袋为安信号"。

### 27.2 撤离税（House Edge 升级 ⑥）

| 项 | 规则 |
|---|---|
| 触发条件 | `cashOut` 路径（已校验 `credits ≥ quota`）；`forceDebtCashOut` 不在此机制内 |
| 计税基 | `surplus = max(0, credits − quota)`（仅超出配额部分） |
| 税率 | `evacuationTax.surplusTaxRate = 0.08`（默认 8%；引擎钳到 [0, 0.5]） |
| 公式 | `tax = floor(surplus × 0.08)`；`payout = credits − tax` |
| 数据写入 | `lastEvacuationTax = { enabled, grossCredits, quota, surplus, rate, amount, payout }`，并随 cashOut 返回值与 listener 第 3 参数透传给 UI |
| 兼容回退 | `evacuationTax.enabled = false` 时 `tax = 0`，payout = grossCredits |

**结果屏手续费行**（仅 `surplus > 0 && amount > 0` 时显示）：

```
手续费    −¥{amount} · 8% · FBC §8.1
```

> 位置：成功撤离 `result-overlay__stat-sheet` 内，"净值"行与"连胜"行之间。CSS 用 `--evtax` 修饰类（暖红降饱和色，dashed 上边界与"被扣"语义对齐）。

> **设计意图**："赚得多交得多"而不是"赚了还扣"——配额内的钱原封不动落袋，只对超额部分征 8%。玩家心理上是"被分了一杯羹"而非"被坑"，与 fbcEdge 的"流通损耗"形成"局内分次抽水（fbcEdge）+ 局末一次性分红（撤离税）"的双层税制。

### 27.3 与战绩页 NET P&L 的对接

| 字段 | v0.06 行为 | v0.07.1 行为 |
|---|---|---|
| `total_credits_collected` | += grossCredits（毛账面） | += payout（已扣撤离税的实际到账） |
| NET P&L 计算 | `collected − buyIn`（毛账面 − 投入） | `collected − buyIn`（**已扣税到账 − 投入**，更精准的"实际净收益"） |
| 兼容旧档案 | 旧档无 v0.07.1 改动；新合并行为只影响 v0.07.1 之后入账的局 | 不需要迁移 |

> **设计意图**：让战绩页"综合损益结算 · NET P&L"严格等于"实际到账 − 实际投入"，把撤离税的成本透明地反映在跨局对账中——玩家如果连续高额撤离，会清楚看到自己的 NET P&L 增速被税率压制，从而催生"局内见好就收"的策略偏好。

### 27.4 RTP 估算（先验估值，待实测校准）

### 28.3 连胜干预·探测协议（pressureProtocol）

**触发流程**：

```text

  → 第 4 局开始时（buyIn 入口），剩余 0 → 协议自然结束
```

**为何 fbcEdge +2% 而不是 +5%**：fbcEdge 基线已经是 1–8% 区间（NEGATIVE 1% / POSITIVE 3% / DOUBLE 5% / GOLDEN 8%）；+2% 是"恰好让玩家感受到'本局抽水比平时狠'"而不是"被一刀砍死"。3 局协议期总共多扣 ≈ 6% 净利——与"3 局连胜的奖励曲线"基本相抵，让玩家"高位横盘"。

**授权面板渲染**（`#deploy-pressure-protocol-line` · 仅 active 时显示）：

```text

```

> **设计意图**：让"连胜"不再只是"运气加成"——3 局连胜后玩家会被庄家"上特别名单"，下 3 局抽水加重。这是"贪心被惩罚"的跨局回路：玩家可以选择激进维持连胜（进入更高税区）或主动降速断连胜（让协议自然消亡）。

### 28.4 高偏移加码（pressureProtocol.corruption）

**触发条件**：`game.corruptionRatio > thresholdRatio`（默认 0.65，即 corruption ≥ 总上限 65%）。判定时机：每层 `processAscendComplete()` 入口——意味着同一局内 corruption 涨过/跌过阈值，加码会逐层启停。

- countdown × 0.9（默认）—— 6000ms 标准 → 5400ms；最低钳到 1500ms 防误关闭决策窗口。

> **设计意图**：高 corruption 等同于"频繁触发 FBC 代决议风险"——玩家本就处于"委员会随时介入"的临界状态，加码 10% 的决策窗收紧不构成"硬伤"，但**通过副标题文字让玩家明确"现在更危险"**，避免"无声的难度提升"。

### 29.4 货舱清点点击跳过

**需求**：熟悉玩家不需仔细阅读每张卡，等待破坏心流；新增点击跳过机制，自由节奏不强制。

**设计意图**：模拟 FBC 终端从虚空中"打印"出数据的质感。

**三层信号**（全部直连 `master`，绕过 BGM 低通滤波器）：

| 层 | 频率 / 类型 | 包络 | 时长 |
|----|-----------|------|------|
| ① 噪底 | 带通白噪 ~2.8kHz | 150ms 淡入 → 保持 → 120ms 淡出 | 440ms |

### 29.6 音效微调：Stamp `playbackRate` 微随机化

**方法**：`playFbcOverrideStamp()` 每次调用生成 `rate = 0.95 + Math.random() × 0.1`（均匀分布，范围 0.95×–1.05×）。

**应用方式**：

| 音层 | 处理方式 |
|------|----------|
| 重落 thud（square oscillator） | 所有 `frequency` 值 × `rate` |
| 纸面摩擦（BufferSource 噪声） | `src.playbackRate.value = rate`（原生 Web Audio API） |
| 阴森余音（sawtooth oscillator） | 所有 `frequency` 值 × `rate` |

**效果**：连续快速盖章时每次音调/速度有 ±5% 微差，消除机械重复感，增加物理真实感。
