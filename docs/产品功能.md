## 二、完整流程事件分析

### 2.1 一局完整流程

```
启动屏 → [首次] 开场幕布 → 许可面板（探测杠杆 / 启动或维持现状）
  │
  └─ [循环 · DECIDING]
        │
        ├─ 上行 → 随机跳 1–8 层（每层基线增长）
        │
        ├─ 乘客下车 / 上车（概率触发；可能弹出对赌 confirm / 天使 buff）
        │
        ├─ 安全屋检查（第 5/10 层首次）→ 净化 or 跳过 → 回到决策
        │
        ├─ 环境事件轮换（每 5 层）
        │
        ├─ 主事件掷骰 → LIQUIDATION / NEGATIVE / POSITIVE / DOUBLE（含黄金正面等子情况）
        │
        ├─ 应急制动检查（若 LIQUIDATION 且已武装 → 缓和留存）
        │
        ├─ 乘客资本修正 + volatile 波动
        │
        ├─ 偏移指数累积（受环境、对赌存续期等调制）
        │
        ├─ 惊喜事件（非清算）：硬币雨 / 跳层
        │
        ├─ 可选：卡牌选取（非清算且启用）→ 翻牌 → 奖励或爆仓类事件 → REVEALING
        │
        ├─ Breach 判定（非清算、达最小楼层）
        │
        ├─ 开门展示（叙事闪屏 → 金额闪屏等 uiPacing）：
        │     ├─ 未缓和的 LIQUIDATION → GAME_OVER
        │     ├─ Breach → 倒计时（超时自动清算）
        │     └─ 否则 → DECIDING（继续 / 撤离；软盘 scan/brake）
        │
        └─ 撤离 → 配额/净资产检查 → 质询（概率）→ 结算
```

### 2.2 收益类事件（资产上升或风险降低）

| # | 事件 | 触发 / 条件 | 参数位置 | 效果摘要 | 代码锚点 |
|---|------|-------------|----------|----------|----------|
| 1 | 基线增长 | 每升一层 | `baselineGrowth` | 资产 × 倍率 | `_applyBaselineGrowth()` |
| 2 | POSITIVE / DOUBLE | 掷骰落入对应带 | `thresholds` · `outcomeBands` | 经 **动态杠杆公式** 增减账面 | `_applyOutcomeToCredits()` |
| 3 | 黄金正面 | 正面带内再掷骰 | `goldenPositive` | 正面倍率升级为超高档（仍 POSITIVE） | `rollEventFromR()` |
| 4 | 硬币雨 / 跳层 | 非清算、`minFloor` 起 | `surprise` | +比例资产 / 额外升层+基线 | `SurpriseEvent` |
| 5 | VIP / 组合加成 | 乘客在场 | `passenger` · `combos` | 资本乘子、阈值修正 | `aggregatePassengerModifiers()` |
| 6 | DANGER 正面 volatile | 60% 分支 | `passenger` | ×`volatilePositiveMult` | `processAscendComplete()` |
| 7 | 天使投资人 | 未伪装 VIP 上车 | `angelInvestor` | 存续层数内抬高正面 rate | `_angelLift` |

| 9 | 连击盲信 | 短窗口内再次上行 | `comboBlindConfidence` | 本层 delta ×`gainMult`（隐性 quota 压力） | `_blindConfidenceActive` |

### 2.4 配置文件说明（v0.03）

`config/game-events.json` 顶层除 `_meta` 外，主要配置节包括（修改后无需改代码即可调平衡）：

| 配置节 | 涵盖内容 |
|--------|----------|
| `thresholds` · `outcomeBands` | 四事件带阈值与各带倍率 |
| `corruption` | 偏移上限、各场景累积、清算带放大上限 |
| `floorLeverage` · `pigPeriod` · `baselineGrowth` · `floorsJumped` | 杠杆、养猪期、基线、跳层 |
| `breach` | Hiss 入侵 |
| `passenger` | 乘客权重、修正、伪装、组合、volatile |
| `surprise` · `envEvents` · `safeNode` · `quota` | 惊喜、环境、安全屋、配额压力 |
| `speculatorProtocol` · `angelInvestor` | 对赌协议、天使增益 |
| `rerollBailout` | 重掷清算买断（依赖引擎 `rerollCurrentFloor` 被调用） |
| `goldenPositive` · `chainLightning` · `assetFreeze` · `comboBlindConfidence` | 肥尾、连环负债、冻结、连击 |
| `suspense` | 清算/负面揭晓阻塞毫秒 |
| `copy` | 配额拦截、对赌、重掷买断等文案模板 |
| `item` | 软盘次数与制动留存比例 |
| `bossChallenge` | 质询触发 |
| `uiPacing` | 叙事/金额/结算等 UI 节拍 |
| `biome` | 场景楼层区间 |
| `cardSelection` | 卡牌选取开关、身份与奖/罚事件池 |

**说明**：`rerollBailout` 与 `onReroll` 相关逻辑已在控制器内闭环，但 **当前 HTML 未提供玩家触发 `rerollCurrentFloor` 的入口**；上线前若需要该玩法，需在 UI 层增加绑定（并消耗软盘或独立规则由产品设计定）。
