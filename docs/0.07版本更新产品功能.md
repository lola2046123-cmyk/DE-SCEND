# 《暴富电梯》0.07 版本产品功能表（House Edge 第二阶段 + UX 打磨）

> **版本**：0.07.3（增量发布，与 0.06.3 完全兼容）
> **生成日期**：2026-04-30
> **里程碑序**：0.07 信用评级 → 0.07.1 浮盈折现 + 撤离税 → 0.07.2 委员会三触发（庄家人格化）→ **0.07.3 心流修复 + UX 打磨**（纯视觉 / 音效 / 交互层，无经济数值变更）。
> **说明**：本版本延续 0.06 的 House Edge 路线，在数值维度新增两道抽水机制（浮盈折现 + 撤离税），在叙事维度引入庄家人格化，在 UX 维度做心流修复和视觉规范统一。所有新机制均可通过 config flag 独立关闭。

## 一、v0.07 — 信用评级系统

### 立项动机

0.06 之后，破产惩罚仍停留在"本局归零"层面。高频破产玩家可以无限重开、继续享受同等起点待遇，缺乏跨局积累的**纵向代价**。信用评级在生涯层面引入持久性惩罚与奖励，为高风险行为附加长期成本。

### 设计规格

**配置**：`config/game-events.json · creditRating`
**持久化**：`localStorage · fbc_career_archive_v1 · fbc_credit_rating`
**代码**：`js/game-controller.js` / `index.html · refreshCreditRatingUI`

| 评级 | 触发条件 | 效果描述 |
|---|---|---|
| AAA | 连续 3 局无破产 + 净盈利 | 联邦优先通道激活：正向 FBC Edge 略减 |
| A | 默认档 | 标准条件 |
| B | 累计 1–2 次破产 | 偏移概率略增 |
| C | 累计 3–4 次破产 | 流动性管理介入：cashOut 附加扣除 |
| D | 累计 ≥5 次破产 | 债务重组期：初始杠杆上限降低 + 动画脉冲警告 |

**UI 落地**：
- 授权面板顶部信用评级行（`#deploy-credit-rating-line`），仅在评级 ≠ A 且 `creditRating.enabled=true` 时显示，颜色按档位差异化（AAA=mint / B=暖珊瑚 / C=弱红 / D=危险红+脉冲动效）。
- 评级文案风格：`// 信用评级：D 级 · 进入债务重组期。`（Courier New 官僚体）。

**回退**：`creditRating.enabled = false` → 整段逻辑 no-op，生涯档案字段留空。

## 二、v0.07.1 — 浮盈折现（Surplus Discount）

### 立项动机

单层 RTP 经 0.06 ①② 压到约 117% 后，"已大幅超过配额"的玩家依然可以按原倍率继续滚雪球。高倍率杠杆叠加大资产，单局账面暴增过快，House Edge 在后期实际失效。浮盈折现模拟"资产过大时局方切一道额外税"，把大资产期的实际 RTP 进一步压缩。

### 设计规格

**配置**：`config/game-events.json · surplusDiscount`
**代码**：`js/game-controller.js · _applyOutcomeToCredits`（在 `fbcEdge` 之前插入）
**UI**：`index.html · showDoorResultFrame` → `#drf-discount` 注脚行

| `stake / quota`（当前资产倍率） | 折现乘子 | 标签 |
|---|---|---|
| < 1.5 | 1.00（无折现） | — |
| < 3.0 | 0.95 | `light` |
| < 6.0 | 0.88 | `heavy` |
| ≥ 6.0 | 0.80 | `extreme` |

**抽水时机**：正向结算 delta 确定后、`fbcEdge` 扣税前，乘以折现乘子；`outcome._surplusDiscountMult` / `_surplusDiscountAmt` 写入供 UI 注脚显示。

**视觉表达**（三位一体帧注脚，仅在折现生效时显示）：
```
// FBC §5.2 浮盈折现 ×0.88 · 已扣 ¥28
```

**回退**：`surplusDiscount.enabled = false` 时整段 no-op。

## 三、v0.07.1 — 撤离税（Evacuation Tax）

### 立项动机

浮盈折现只压单层 delta，但玩家可以通过累积多层后一次撤离来绕开单层税。撤离税在 `cashOut` 出口处对**超额盈余部分**征收，使"超额撤离"的实际到手金额低于账面。

### 设计规格

**配置**：`config/game-events.json · evacuationTax`
**代码**：`js/game-controller.js · cashOut` 末端
**UI**：结果面板手续费行（`result-overlay__stat-row`）

**征税逻辑**：
```
surplus = max(0, credits - quota)
tax     = floor(surplus × evacuationTax.rate)
payout  = credits - tax
```

仅对 `surplus > 0` 的情况征税；`quota` 以内的部分不征税，保护低盈利玩家。

**视觉表达**（结果屏，仅盈余 > 0 时显示）：
```
撤离税（超额 ×12%）  −¥420
```

`lastOutcome.evacuationTax` 写入，供结算层和战绩系统引用。

**回退**：`evacuationTax.enabled = false` → 跳过征税，`payout = credits`。

## 五、v0.07.2 — 委员会三触发（庄家人格化）

### 立项动机

0.07.1 已在数值层建立压力，但叙事层缺乏反馈。玩家大赢、连胜、高偏移时，"委员会"无任何反应，沉浸感断裂。三触发机制让委员会作为"庄家人格"对高风险局势主动发言，强化叙事张力。

### 触发一：大赢反应（Big Win）

**触发条件**：单层 `delta > quota × 4`
**行为**：`finishReveal()` 中监听结果，满足条件时随机从 `COMMITTEE_PRESSURE_LINES.bigWin` 池抽一条低语台词，通过委员会弹窗或场景覆层展示。

**台词风格示例**：
```
「有趣——这一手，委员会记住了。」
「数字越漂亮，离出口越远。」
```

### 触发二：连胜探测协议（Consecutive Streak）

**触发条件**：连续 N 层正向结算（config 可配）
**行为**：激活"探测协议"，在生涯档案写入 `pressure_protocol_remaining_runs`；在此期间 `pickFbcEdgeRate` 额外加 **+2%** 提升抽水力度（叙事语义：委员会介入加码）。

**UI**：授权面板顶部"探测协议行"（`#deploy-pressure-protocol-line`），与信用评级行可同时显示，竖向堆叠，使用冷蓝官僚色以区别于信用评级的暖色系。

### 触发三：高偏移加码（High Corruption）

**触发条件**：当前偏移指数超过阈值（config 可配，默认 ≥ 70%）
**行为**：`prepareLockerHand` 中 `countdownMs × 0.9`（决策时间压缩 10%）；选盘倒计时副标题切换为"高偏移区 · 系统已介入"的官僚风文案。

**叙事逻辑**：高偏移时玩家判断力降低，委员会乘虚施压，在视觉节奏上制造压迫感。

**回退**：三触发均可通过 config flag 独立关闭，不影响状态机和核心回路。

## 六、v0.07.3 — 心流修复 + UX 打磨

### 6.1 战绩缺省图（Empty Career State）

**问题**：战绩页空状态仅展示一行纯文字"// 暂无作业记录 · 上行后自动归档"，视觉密度过低，初见玩家不清楚该空间的含义。

**改动**：`index.html · renderCareer` 中注入 CSS 美术电梯井图示，风格与 Icons 规范一致（线性向量、单色系），由以下层级组成：
- 竖向井道（`__shaft`）+ 左右导轨列
- 电梯轿厢（`__cabin`）+ 可动门（`__door`）+ 顶栏/底栏
- 楼层点状指示灯带（`__floors`），部分点亮以示"有楼层可上行"
- 徽章层（`__badge`）覆盖空文案

配合 `@keyframes fbc-empty-fadein` 淡入，无额外 DOM 改动。

### 6.2 全局财务数值颜色翻转（中国金融市场规范）

**触发反馈**：玩家反馈 UI 正负颜色与中国股市/金融 App 习惯相反（原方案绿=正/红=负，与国内用户心智模型冲突）。

**新规范**：**红色 = 正向 / 上涨（+、收益）；绿色 = 负向 / 下跌（−、亏损）**

| 位置 | 旧 | 新 |
|---|---|---|
| HUD 净资产 `--negative` / `--positive` | 绿涨红跌 | 红涨绿跌 |
| 货舱清点卡片极性色 `gain` / `loss` | 绿 / 红 | 红 / 绿 |
| 资产裂变（交易收据）`gain` / `loss` | 绿 / 红 | 红 / 绿 |
| 战绩 NET P&L `pnl-gain` / `pnl-loss` | 绿 / 红 | 红 / 绿 |
| 菜单·规则页"货舱清点"颜色描述 | 绿光为正向 / 红光为损失 | **红光为正向（收益）/ 绿光为损失** |

> 情报员标签 `INTEL ✓`（绿）/ `WARN !`（红）是游戏内**方向性情报标记**，与财务颜色体系无关，保持不变。

### 6.3 货舱清点心流修复（Manifest Reveal 时序压缩）

**问题**：货舱清点结束后，玩家等待操作按钮重新出现的时间长达约 8–9 秒，严重破坏心流。根因：

1. **双重等待**：`runManifestReveal` 返回 `totalMs`，调用方等待 `totalMs + doorRevealSettleMs` 后才调用 `finishReveal()`，两个定时器串联叠加。
2. **参数过大**：各节拍默认值（`manifestItemRevealMs = 1500ms` / `manifestLastExtraHoldMs = 1200ms` 等）偏保守。

**修复方案**（架构 + 参数双管齐下）：

架构重构：`runManifestReveal(o, delta, biome, locker, onHide)` 接受 `onHide` 回调，在 overlay **开始淡出的同一帧**（而不是淡出结束后）触发 `_doFinishReveal()`，消除 ~1.2s 死窗。

参数压缩（`config/game-events.json · uiPacing`）：

| 参数 | 旧值 | 新值 | 说明 |
|---|---|---|---|
| `doorRevealSettleMs` | 200 ms | 0 ms | 三位一体帧出现与清点起步并行 |
| `manifestItemRevealMs` | 1500 ms | 820 ms | 单张持读时间 |
| `manifestLastExtraHoldMs` | 1200 ms | 360 ms | 末张额外停留 |
| `manifestCardOutMs` | 280 ms | 260 ms | 出场动画 |
| `manifestBgOutMs` | 360 ms | 320 ms | 背板消隐 |

**效果**：总等待从 **~8.16 s** 压缩至 **~3.28 s**（含背景音切换），保留足够阅读时间的同时恢复心流节奏。

### 6.4 货舱清点点击跳过

**需求**：熟悉玩家在卡片停留（holding）阶段无需细读，希望点击快进。

**实现规格**：
- 持留阶段 overlay 区域变为可点击（`cursor: pointer`）
- 提示文字切换为 `// 点击任意处继续`
- 点击后：
  - 播放 `audioEngine.playManifestRevealItem('neutral')` 作为点击反馈
  - 非末张 → 立即切换到下一张卡片
  - 末张 → 立即触发 `triggerHide()`（等同于自然结束）
- 进入阶段（entering）不可跳过，防止误触导致卡片展示不完整

**关键变量**：
- `_manifestLive`：追踪当前卡片索引与阶段（`idle / entering / holding / exiting / skipping`）
- `_manifestSkipHandler`：点击监听句柄引用，用于精准 `removeEventListener`
- `_manifestOnHide`：模块级回调注册（见 6.5）

### 6.5 流程卡死 Bug 修复

**Bug 描述**：点击跳过机制上线后，货舱清点结束后操作按钮不出现，流程卡死（所有楼层均有，高层更易触发）。

**根因**：`showDoorResultFrame()` 会在 4000ms 后自动触发 `hideDoorResultFrame() → hideManifestStrip() → _clearManifestTimers()`，导致 `triggerHide` 的 setTimeout 被取消，`onHide` 回调永远不触发，`game.finishReveal()` 永远不执行，状态机停在 `REVEALING`，按钮始终 disabled。

**修复**：新增模块变量 `_manifestOnHide` 存储当前 onHide 回调：
- `runManifestReveal` 启动时：`_manifestOnHide = onHide`
- `triggerHide()` 执行时：先清 `_manifestOnHide = null`（防重复），再调 `onHide()`
- `hideManifestStrip()` 末尾：若 `_manifestOnHide` 仍有值（即外部强制中断路径），立即调用并清零

三段保险确保无论何种退出路径（自然结束 / 跳过末张 / DRF 4s 超时 / 游戏重置），`finishReveal()` 有且仅被调用一次。

### 6.6 音效新增：三位一体展示框扫描音（`playScanPrint`）

**绑定动作**：三位一体帧（`#door-result-frame`）淡入时调用。

**音色设计**（全部绕过 BGM 低通滤波器，直连 `this._master`）：

| 层 | 合成方式 | 参数 | 时长 |
|---|---|---|---|
| 噪声底层 | 白噪声 + 带通滤波 2.8 kHz | 淡入淡出包络 | 440 ms |
| 扫描线 | 正弦波 600 Hz → 960 Hz | 频率爬坡 + 淡入淡出 | 420 ms |
| 锁定点击 | 方波 1180 Hz → 720 Hz | 极短衰减 | ~50 ms |

**交互规范**：即使在"音频降维（沉浸模式）"开启时，所有 UI 音效仍保持清晰锐利穿透力。

### 6.7 音效微调：Stamp 音 playbackRate 微随机化

**动机**：连续点击 Stamp 音效完全一致，听感机械。

**改动**：`playFbcOverrideStamp()` 中加入 `rate = 0.95 + Math.random() * 0.1`，每次调用随机 **0.95×～1.05×** 播放速率，振荡器频率与 BufferSource 同步缩放。

### 6.8 授权面板行距收缩（移动端首屏优化）

**问题**：主流手机竖屏（360–430×640–844px）下，「启动电梯」按钮需向下滚动才可见，影响第一帧体验。

**改动策略**（CSS Only，不动 DOM/JS）：

1. **基础层**：减小 `.risk-auth-panel__inner` 的 `gap`、规程折叠区内边距、行高（`line-height` 1.75→1.52 等），信用评级/探测协议行上边距收紧。
2. **短屏断点** `max-height: 760px` / `640px`：在原有收紧规则上再压一档，减小 `deploy-panel` 的竖向 padding 和各子块 gap。
3. **新增断点** `@media (max-width: 480px) and (max-height: 820px)`：针对常见竖屏手机（宽 ≤480px 且高 ≤820px）做最激进收缩：内层 gap 14px、输入框/预设键高度 44px、CTA 间距 28px，**主按钮 min-height 保持 44px** 以满足触控规范。

---

### 6.9 菜单弹窗优化

#### Tab 等高修复

**问题**：「规则」与「战绩」tab 内容高度不同，切换时面板整体跳高/跳矮，视觉抖动。

**改动**：`.fbc-menu-panel` 改为固定高度 `height: min(86vh, 580px)` + `min-height: 360px`（原为仅 `max-height: 86vh`），两个 tab 页面均为 `display: flex; flex-direction: column; min-height: 100%`，始终充满同一容器。

#### 规程折叠抖动修复

**问题**：点击「规程要点」展开/收起时，整个面板内容区上下跳动。

**根因**：`.deploy-panel` 用 `justify-content: safe center`，`<details>` 高度变化时 flex 重新算垂直居中，导致整块文案上移/下移。

**修复**：
- `justify-content: safe center` → `justify-content: flex-start`（`.deploy-panel` 与 `.risk-auth-panel__inner` 同步修改）
- `scrollbar-gutter: stable` 防止滚动条出现时内容区宽度突变引发二次排版抖动
- `summary::after` 改为 `inline-block + min-width: 1.1em`，▼/▲ 切换不引起行宽微变

#### 规则文案更新

| 章节 | 变更内容 |
|---|---|
| 货舱清点 | 颜色描述与全局规范对齐；新增「阅读期间点击任意空白区域可跳过，进入下一张」操作提示 |
| 结算修正（新增） | 浮盈折现（资产超配额越多抽水越重）+ 撤离税（超额盈余征收比例）的玩家提示 |
| 音频/操作 | 精简，去掉与授权面板重复的规程说明 |

| 高倍率区（surplus > quota×3）平均到手 | ≤ quota × 2.5 | > quota × 4 说明折现强度不够 |
| 货舱清点点击跳过率 | 15–40% | > 60% 说明默认阅读时间过长，可继续压缩 |
| 流程卡死（按钮不出现）反馈 | 0 | 任何出现均需优先排查 |
| D 级信用玩家留存率 | ≥ 50% | < 30% 说明惩罚过重 |
