> **用途**：集中列出玩家可见或辅助无障碍读屏的文案，便于统稿、翻译与合规审查。  
> **维护**：改文案时优先对照下表「来源」列；配置项以 `config/game-events.json` 为准，代码内联为兜底或硬编码。

## 15. 开门叙事闪屏（引擎 · `GameController.getOutcomeNarrative`）

数据：**B** · `OUTCOME_NARRATIVE_POOL`（按结果类型随机 `tag` + `text`）

- **DOUBLE**：资产裂变 / 意外分红 / 财富跃升（各条见代码）  
- **POSITIVE_HIGH / LOW**：财富跃升、盲盒大奖、资产裂变、意外分红、特别津贴  
- **GOLDEN_FLOOR**：黄金楼层、财富跃升  
- **NEGATIVE**：恶意做空、违规罚款、通货膨胀  
- **LIQUIDATION**：强制清算  

覆层展示格式：`[ {tag} ]` + 正文（见 `showNarrativeFlash`）。

## 21. 已知文案分叉（供后续统一时参考）

1. **配额未达成**：`copy.quotaNudge` 与 `showDebtWarning` 默认正文均为局内划扣口径；若配置注入更长模板，宜与许可脚注「信用折偿」一致。  
2. **叙事双轨**：引擎 `getOutcomeNarrative`（B）与场景 `formatOutcomeNarrativeHTML`（A）主题相近、句式不同。  
3. **对外品牌**：主标题「暴富」仅用于启动/舞台品牌区；局内术语仍以配额、清算、归档为主，避免全文堆叠口语。  
4. **权力制衡 vs 身份卡**：产品文档称「权力制衡」，UI 仍写「选一张身份卡」；若统一口径需改 **A** 中 `#card-selection-overlay` 文案并回写本节 §8。  
5. **BGM 控件**：无障碍属性为「BGM 降维」；面向用户文档可称「沉浸模式」，翻译键建议以 `aria-label` / `title` 为法律与读屏锚点。
